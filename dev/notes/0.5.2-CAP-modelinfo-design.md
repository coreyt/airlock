# 0.5.2-CAP-modelinfo — design memo

## Goal
Per-model capability metadata on `GET /model/info`, **computed** at startup and
injected into the in-memory config before the LiteLLM child reads it. Single
source of truth: `airlock/capability.py`. No hand-written `model_info:` in
`config.yaml`.

## New pure helpers in `airlock/capability.py`
- `endpoints_for(entry: dict) -> list[str]`: start `["chat"]`; append `"batch"`
  iff `entry.get("airlock_batch")` is truthy OR the entry is a `vertex_ai/`
  model (`litellm_params.model` startswith `"vertex_ai/"`) with a **regional**
  `vertex_location` (set and `.lower() != "global"`). Bare `vertex_ai/` with
  `global` location → `["chat"]` (codex-BLOCK-3 anti-overclaim rule).
- `capability_record(entry: dict) -> dict`: returns exactly
  `airlock_provider` (reuse `airlock_provider_for`), `endpoints`, `underlying`
  (`litellm_params.model`), `region` (`litellm_params.vertex_location` or None),
  `deprecated` (`model_name` endswith `("-aistudio","-vertex","-batch")`).
  Empty/missing `litellm_params.model` must not crash.

Pure; no `proxy` import.

## Injection seam — `proxy._prepare_runtime_config`
After config dict load, unconditionally iterate `model_list`; for each dict
entry: `entry.setdefault("model_info", {}).update(capability_record(entry))`
and set `changed = True`. Preserves pre-existing `model_info`. Import
`capability_record` from `airlock.capability` (pure, no cycle).

## Tests (RED first)
- `tests/test_capability.py`: endpoints_for + capability_record cases incl.
  vertex global → chat-only, regional → batch, suffix-twin deprecated.
- `tests/test_proxy.py`: drive `_prepare_runtime_config` on a temp config with a
  batch-marked + vertex-global entry + a pre-existing model_info; assert
  computed model_info on every entry and preservation.
