# 0.5.2-DOCS — Doc-classification plan (provider-explicit naming + capability discovery)

Release docs for the 0.5.2 work: `provider/model` aliases across the catalog,
per-model capability on `GET /model/info` + an additive `airlock` object on
`GET /v1/models`, and the `X-Airlock-Served-By` verify surface. Per
`dev/update-docs.md` (dev-docs = source of truth → `docs/` derived).

## Facts verified against the shipped tree (`4610db0`)

- `config.yaml` — `provider/model` aliases dual-listed for the whole catalog
  (`anthropic/`, `openai/`, `aistudio/`, `vertex/`, `mistral/`, `perplexity/`,
  `tavily/`, `vllm/`). Batch markers (`airlock_batch`) ride only on:
  `aistudio/gemini-3.5-flash`, `aistudio/gemini-3.1-pro`, `mistral/mistral-large`,
  `mistral/mistral-small`, `vllm/qwen3.6-27b` (and the legacy twins).
- **Vertex is chat-only as shipped:** `vertex/gemini-3.5-flash` and
  `vertex/gemini-3.1-pro` have NO `airlock_batch` marker and `vertex_location:
  global` → `endpoints: [chat]`. `airlock/capability.py:endpoints_for` appends
  `batch` ONLY for a truthy `airlock_batch` OR a `vertex_ai/` model with a
  *regional* (non-`global`) `vertex_location`. Do NOT document Vertex batch at
  `global`.
- `airlock/capability.py:capability_record` returns exactly
  `airlock_provider`, `endpoints`, `underlying`, `region`, `deprecated`.
  `deprecated` is `model_name.endswith(("-aistudio","-vertex","-batch"))` — i.e.
  ONLY the suffix twins carry `deprecated:true`; bare names do not.
- Served-by token mapping (`airlock/capability.py:normalize_provider_token`):
  `aistudio → gemini`, `vertex → vertex_ai`; native tokens pass through. So the
  discovered `airlock_provider` equals the post-call `X-Airlock-Served-By`.
- Surfaces: `/model/info` merges the record into `model_info`
  (`airlock/proxy.py:171`); `/v1/models` folds it under an `airlock` object
  (`airlock/models_seam.py:170`). The `airlock` object is purely additive
  (OpenAI-compat preserved).
- `X-Airlock-Served-By` / `-Region` already cataloged in
  `docs/reference/response-headers.md` + `docs/guide/observability.md`; emitted
  by `airlock/transparency.py:298`.
