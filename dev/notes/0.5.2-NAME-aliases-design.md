# 0.5.2 NAME-aliases — implementation design memo

Provider-prefixed `model_name` aliases for the whole catalog + collision-safe
alias resolution + a shared provider classifier. Self-contained pack.

## Approach

### A. config.yaml (root) — dual-listed prefixed aliases
Add a sibling `model_name` entry (same `litellm_params`) next to each existing
entry per the Appendix-A map. Consolidated batch entries (`aistudio/gemini-3.5-flash`,
`aistudio/gemini-3.1-pro`, `mistral/mistral-large`, `mistral/mistral-small`,
`vllm/qwen3.6-27b`) carry the `airlock_batch` marker copied from their legacy twin.
Legacy twins, `fallbacks`, `cost_tiers`, `smart`, `files_settings` untouched.

### C. airlock/capability.py (NEW) — shared classifier
- `normalize_provider_token(token)`: `aistudio`->`gemini`, `vertex`->`vertex_ai`,
  `vertex_ai_beta`->`vertex_ai`, identity otherwise. ONE source for §B + §C.
- `airlock_provider_for(entry) -> str | None`: served-by token from
  `litellm_params.model` prefix, normalized; `enhanced/<profile>` resolves through
  `enhanced_profile.target_model` -> `gemini`. Pure, no I/O. No import of router.

### C. router.set_router_config
Build `_alias_provider_map[model_name] = airlock_provider_for(entry)` (skip None).
`infer_provider` catalog-first logic + `_PROVIDER_PREFIXES` fallback unchanged.
Do NOT add aistudio/vertex to `_PROVIDER_PREFIXES`.

### B. model_alias.py — collision-safe loader + provider-aware strip
Two-pass `load_from_config`:
- Pass 1: insert every explicit `model_name` key (authoritative, immutable).
- Pass 2: add bare/version-stripped variant keys only when not an explicit key AND
  claimed by exactly one entry (drop ambiguous, never last-write-wins).

Provider-aware index built at load time:
- `_provider_body_alias[(token, body)] -> alias` (first-writer wins per token+body).
- `_body_providers[body] -> set(tokens)`.
body = stripped litellm-model body + version-stripped form; token =
`airlock_provider_for(entry)`.

`resolve()`:
1. exact full-string match (explicit + unambiguous variants) — unchanged.
2. slash miss: `prefix, bare = split`; `token = normalize_provider_token(prefix)`.
   - `(token, bare)` in index -> resolve (cache).
   - else bare in `_body_providers`: single provider -> resolve regardless of
     prefix; multi-provider (prefix did not disambiguate) -> return None (no fuzzy,
     no cache).
   - else bare in `_exact` -> resolve (cache).
3. non-slash unknowns / unknown bare -> fuzzy as today.

  AI Studio; vertex/ + vertex_ai/ -> vertex; aistudio/ + gemini/ -> AI Studio;

  no fuzzy/cache; legacy -aistudio/-vertex resolve to self).

  enhanced/ -> gemini, vllm -> openai.

  targets live) + batch-keys (load_batch_aliases has new consolidated + legacy keys).
