# 0.5.4-MIGRATE-entfathom-cutover (2b-ii) — design memo

Behavior-preserving CUTOVER: the `RequestRecorder` becomes the single live telemetry
callback in enterprise's old slot; enterprise + fathom become pure sinks; the old
duplicated builders are deleted. The live logged/served surface is field-for-field
unchanged except the already-registered `timestamp` convergence (design §3.4).

## What changed

1. **Oracle freeze (step 1).** `tests/test_projections_equiv.py` previously compared
   `project_*` against the LIVE `_build_record`/`_fathom_properties`. Those builders
   are deleted here, which would destroy the oracle. Captured their output once (with
   `AIRLOCK_*` cleared to match `conftest.clean_env`; `_FakeResponse.__repr__` made
   deterministic so `_serialize(resp)` is process-stable) into
   `tests/fixtures/0.5.4-entfathom-golden.json` and rewired the tests to assert
   `project_enterprise/fathom(event)` == frozen golden (timestamp stripped, datetimes
   normalized). The equivalence proof no longer depends on the live builders.

2. **Activation (step 2).** `airlock/callbacks/recorder.py` now builds the recorder via
   `_build_recorder()` (enterprise sink always-on FIRST; fathom sink registered
   async-only ONLY when `AIRLOCK_ENABLE_FATHOM_LOGGER` is set) and `_self_register()`
   installs `recorder_callback` into all four LiteLLM callback lists at import.
   `config.yaml` success/failure_callback → `[recorder.recorder_callback,
   monitor.proxy_monitor]` (recorder FIRST). `proxy.py` no longer appends the fathom
   callback to config (the recorder owns fathom, gated by the same flag).

   **Ordering invariant:** monitor sets `metadata["airlock_provider_protection"]` in its
   `log_failure_event` (monitor.py:326-339). Because the recorder runs FIRST (config
   order + import self-register append order) and snapshots `guardrail_meta` at event
   build time, enterprise's record can never gain that key — pinned by
   `test_recorder_cutover.py::test_event_snapshot_excludes_post_build_provider_protection`.

3. **Deletion (step 3).** Enterprise `_build_record` + `log_*_event`/`async_*` + module
   `_self_register`; fathom `_log_event`/`_base_record`/`_fathom_properties` +
   `log_*_event`/`async_*` + `_self_register_async` (and the now-dead `_debug_enabled`).
   Kept: both sinks' `record_event`, all log helpers, the precall-block/batch/admin
   writers, dedup/engine, rotation/cleanup/redaction, and the lowest-cost None guard
   (now called directly at import).

4. **Test adaptation (step 4 + fan-out).** Legacy builder/callback tests re-pointed
   through the live path (`project_enterprise(build_request_event(...))` and
   `record_event`) via thin in-file shims — assertions unchanged, so the SAME emitted
   record is still pinned (and additionally locked by the frozen goldens). The two
   enterprise `async_log_*_delegates` tests (deleted plumbing now owned by
   `RequestRecorderCallback`) were removed; coverage lives in
   `test_request_event.py`/`test_recorder_wiring.py`.

5. **Seam tests (step 5).** `tests/test_recorder_cutover.py`: snapshot-immutability +
   config ordering, no-double-emit (+ old loggers not separately registered), fathom
   gating (flag/async-only/skip), failure isolation, live install.

## Out-of-scope fan-out (necessary consequences of the cutover)

These files reference deleted symbols / the old dormant contract; the cutover's own
directives necessitate updating them to keep the full no-network suite green. Each was
re-pointed mechanically (no assertion weakened):

- `tests/test_recorder_wiring.py` — `test_recorder_module_is_dormant` → `_is_live`.
- `tests/test_proxy.py` — `test_prepare_runtime_config_can_enable_fathom_logger` →
  asserts proxy no longer injects the fathom callback (recorder owns it).
- `tests/harness/test_phase4_logging.py`, `tests/test_integration.py`,
  `tests/test_ratelimit_observability.py` — re-pointed `_build_record`/`log_*_event`.

## Known pre-existing failure (NOT this pack)

`tests/test_fathom_init.py::test_init_engine_with_fathomdb` fails because the optional
`fathomdb` package is not installed (`init_engine` in untouched `datastore.py` returns
None). Independent of this pack.
