## Approach

New module `airlock/callbacks/request_event.py`:

1. `RequestEvent` ‚Äî `@dataclass(frozen=True)`, the **superset** of every sink's
   fields (identity/lifecycle, request, raw `response_obj`, usage, both failure
   forms, enrichment, transparency). Not pydantic ‚Äî matches `transparency.py`.
2. `build_request_event(kwargs, response_obj, start_time, end_time, *, success)` ‚Äî
   the single producer. **Sources once**, reusing the enterprise helpers verbatim
   (`_normalize_failure`, `_get_airlock_client`, `_serialize`) and the upstream
   helpers `_build_record` already imports (`infer_provider`,
   `attribute_served_backend`, `is_gemini_provider`, `classify_gemini_response`,
   `build_gemini_response_headers`). Mirrors `_build_record` (enterprise_logger.py
   :406-539) field-for-field for the shared fields. Runs the Gemini **enrich-once**
   block (¬ß3.5) that mutates `metadata` in place **before** snapshotting
   `guardrail_meta`. Carries the **raw** `response_obj` (¬ß3.10). Adds the three NEW
   fields: `bare_exception_error` (¬ß3.9), `request_headers` + `mcp_arguments`
   (¬ß3.8).
3. `RequestRecorder` ‚Äî the seam (¬ß5/¬ß5a): ordered sink registry
   (`register(sink, *, name)`, append ‚Üí dispatch in registration order),
   `dispatch(event)` wraps **each** sink in `try/except Exception` (log
   `warning(..., exc_info=True)`, continue), never propagates, empty = no-op,
   `dispatch` itself never raises. Introspection via `sink_names`/`sinks`. NOT
   installed into the live litellm callback manager (deferred to MIGRATE-enterprise).

## Sourcing parity (mirror `_build_record`)

- `lp = kwargs.get("litellm_params", {}) or {}`; `metadata = lp.get("metadata", {}) or {}`.
- `airlock_client = _get_airlock_client(metadata, kwargs)`.
- failure triple from `_normalize_failure(kwargs, response_obj)` iff `not success`.
- usage via `getattr(u, ..., 0)` exactly as the builder.
- `provider = metadata.get("airlock_provider") or infer_provider(model)`.
- Gemini enrich-once ‚Üí recompute `guardrail_meta` AFTER (mutates metadata in place).
- `mcp_meta` block (call_type/mcp_tool_name/mcp_server_name) same conditions.
- mutations: asdict if dataclass else `_serialize`.
- served: `attribute_served_backend(...)` wrapped try/except ‚Üí None; asdict;
  `attribution = "served" if served and served.get("provider") is not None else "inferred"`.

## NEW fields

- `request_headers = kwargs.get("headers")` (raw ‚Äî fathom `headers_json`).
- `bare_exception_error = str(kwargs.get("exception")) if not success else None`
  (s3/sql raw error; note `str(None) == "None"`).
- `mcp_arguments = kwargs.get("mcp_arguments") or lp.get("mcp_arguments") or metadata.get("mcp_arguments")`.

## Test plan (`tests/test_request_event.py`, no network)

In-process fakes: a `_FakeUsage`/`_FakeResponse` (`.usage`, `.choices`), plain dict
kwargs/litellm_params/metadata. Eight cases per the prompt:
1. success field-sourcing (incl. raw `response_obj` identity).
2. failure: rich triple + `bare_exception_error`; `exception=None` ‚Üí `"None"`.
3. NEW fields: `request_headers`, `mcp_arguments` chain, `mcp_meta`.
4. Gemini enrich-once: metadata mutated in place + `guardrail_meta` reflects it
   (monkeypatch the gemini helpers to stay no-network).
5. mutations/served/attribution (stub `attribute_served_backend` to a value and a raise).
6. recorder ordering.
7. recorder failure isolation (raising sink contained; others run; logged; no raise).
8. empty recorder no-op.

## Scope discipline

New module + tests only. No existing sink/registration/config edit. Anchor drift
recorded in `output.json.additional_changes_made_in_scope`. 3-iteration GREEN cap.
