# 0.5.1-SET-loader — design memo

One typed `AirlockSettings` loader: the single in-place reader for every `fast/`
setting, uniform `env > config > default` precedence with malformed-input
fallback. Additive + behavior-preserving — constructed at startup but NOT yet
consumed (SET-unify migrates the router/monitor/breaker consumers and fixes the
R6 nesting bug). Models its structure on `transparency.TransparencyConfig` and
its precedence ladder on `router._load_cost_tiers`.

## `AirlockSettings` (frozen dataclass) fields

| field | type | source (config) | env var | default |
|-------|------|-----------------|---------|---------|
| `provider_budgets` | `dict[str,float]` | `router_settings.provider_budget_config` (via `BudgetConfig`, `budget_limit→max_budget`) | `AIRLOCK_PROVIDER_BUDGETS` (JSON dict) | `_DEFAULT_PROVIDER_BUDGETS` (anthropic/openai 50, gemini 0, mistral/perplexity 25) |
| `budget_windows` | `dict[str,float]` | same block `time_period` via `duration_in_seconds` | (n/a — env budgets carry no window → default window per provider) | 86400 (`"1d"`) per provider |
| `failover_map` | `dict[str,list[str]]` | `router_settings.fallbacks` (list-of-1-key-dicts → dict) | `AIRLOCK_FAILOVER_MAP` (JSON) | `_DEFAULT_FAILOVER_MAP` |
| `cost_tiers` | `dict[str,list[str]]` | top-level `cost_tiers` | `AIRLOCK_COST_TIERS` (JSON) | `_DEFAULT_COST_TIERS` |
| `session_ttl` | `int` | `airlock_settings.session_ttl` | `AIRLOCK_SESSION_TTL` (int) | 3600 |
| `smart_thresholds` | `tuple[float,float]` | `airlock_settings.smart_thresholds` | `AIRLOCK_SMART_THRESHOLDS` (JSON 2-list) | (0.30, 0.60) |
| `budget_warn_ratio` | `float` | `airlock_settings.budget_warn_ratio` | `AIRLOCK_BUDGET_WARN_RATIO` (float) | 0.8 |

Defaults are reproduced verbatim from the read-only anchors (router.py:42-76,123;
monitor.py:39; circuit_breaker.py:32). No existing default is deleted.

## Precedence (uniform, per field)

`env > config > default`. Malformed env → warn, fall back to config. Malformed
config → warn, fall back to default. `0` budget is a real configured value and is
preserved (never coerced to a default) — `0 ⇒ no enforcement/unlimited/no swap`.

## Budget parsing (LiteLLM primitives)

`from litellm.types.utils import BudgetConfig` and
`from litellm.litellm_core_utils.duration_parser import duration_in_seconds`
(verified importable, litellm 1.89.0). For each provider entry under
`router_settings.provider_budget_config`, build `BudgetConfig(**entry)` which maps
`budget_limit→max_budget` and tolerates the native `max_budget`/`time_period`
spelling; `duration_in_seconds(time_period or "1d")` gives the window. Malformed
entry → skip that provider (treat as no budget), warn, continue. `budget_limit: 0`
parses to `max_budget=0.0` and is preserved.

## `fallbacks` shape

`router_settings.fallbacks` is a list of single-key dicts
`[{claude-opus: [...]}, ...]`. Flatten to `dict[str, list[str]]`. Non-conforming
entries skipped with a warning.

## Startup wiring

`configure_settings(config)` mirrors `configure_transparency`: module-level
`_configured` singleton + `get_settings()` accessor (returns defaults if unset).
Called in `proxy.py` startup wiring block alongside the other `configure_*`.

## Test matrix (`tests/test_fast_settings.py`)

Per field: default (no env/no config), config override, env override, env-beats-
config, malformed-env-falls-back-to-config, malformed-config-falls-back-to-default.
Plus: `0` budget preserved as `0`; `fallbacks` list-of-dicts → dict; `"1d"` →
86400 window; native `max_budget` spelling tolerated; `configure_settings`/
`get_settings` singleton round-trip + defaults-when-unset.
