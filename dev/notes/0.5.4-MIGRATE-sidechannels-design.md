# 0.5.4-MIGRATE-sidechannels — design memo

Last MIGRATE pack: feed the per-request Prometheus metrics (and the mutation
counter) from the shared `RequestEvent`/recorder seam, deleting metrics' own
LiteLLM callback methods. Behavior-preserving — the counters emitted per request
are identical to before.

## What changed

### `airlock/callbacks/metrics.py`
- Added `AirlockMetricsCallback.record_event(self, event)`:
  - `requests_total{model,user,success}` on BOTH paths (`success="true"` /
    `"false"`); `user = event.user or "unknown"` (metrics' own default — loggers
    leave `user` None, metrics does not).
  - `request_duration{model}` success-only, computed from raw
    `event.start_time`/`event.end_time` (`.total_seconds()`), NOT `duration_ms`.
  - mutation counting success-only via `_record_mutations_from_event`.
- KEY SUBTLETY: `event.mutations` is the `dataclasses.asdict`'d list (dicts with
  `field`/`op` keys), not raw `Mutation` objects. Replaced
  `_record_mutations(metadata)` (`getattr`-based) with
  `_record_mutations_from_event(mutations: list)` using dict access
  (`m.get("field")`/`m.get("op")`), same skip-when-None and no-crash-on-noniterable
  rules — behavior-identical.
- DELETED `log_success_event`/`log_failure_event`/`async_*` callback methods and
  `_self_register` (def + module-level call). metrics is no longer self-registered
  into LiteLLM's callback lists; it is dispatched via the recorder.
- UNTOUCHED: standalone guardrail helpers (`record_pii_redaction`,
  `record_keyword_block`, `record_threat_block`, `record_response_scan_detection`,
  `record_provider_ratelimit_headroom`, `set_circuit_breaker_state`).

### `airlock/callbacks/recorder.py`
- `_build_recorder` now registers `metrics_callback.record_event` UNCONDITIONALLY
  (always-on, normal success+failure sink) right after the enterprise sink, before
  the env-gated fathom/s3/sql sinks. Order is irrelevant — metrics reads no
  monitor-mutated field.

## Tests
- NEW `tests/test_migrate_sidechannels.py`: requests_total both paths + "unknown"
  default; request_duration success-only (incl. no-times / failure cases);
  mutations from asdict'd event dicts (skip missing field/op; none on failure);
  always-on recorder registration; `_self_register` gone; `metrics_callback`
  absent from all four LiteLLM lists; single dispatch increments once.
- Adapted `tests/test_metrics.py`: re-pointed the callback + mutation tests through
  `record_event` (SimpleNamespace events with asdict'd mutation dicts); removed the
  `_self_register` test (symbol deleted). Assertions preserved, never weakened.
- Adapted `tests/test_recorder_cutover.py` sink-composition assertions to include
  the new always-on `"metrics"` sink (`["enterprise","metrics","fathom"]` /
  `["enterprise","metrics"]`); intent preserved (no double-emit; fathom flag-gated).

## Pre-existing, out of scope
