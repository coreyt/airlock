## R6 fix

Monitor's budget source becomes `get_settings().provider_budgets`, which reads
`router_settings.provider_budget_config` (correct nesting). The old top-level read
(`config.get("provider_budget_config")`) was always empty against the real config shape.

## AC-0 contract (`0 ⇒ no enforcement`, all three layers)

`get_settings().provider_budgets` returns `0.0` for a `0`-configured provider and the key
is absent for an unconfigured one. Both consumers short-circuit on falsy:
- router `_apply_budget_awareness`: `if not budget_limit: return model, None` (no swap)
- monitor `_maybe_warn_budget`: `if not limit: return False` (no warn)
- LiteLLM: skips the hard block on falsy `max_budget` (documented, not code here).

## Test plan (RED first, committed alone)

2. **R2 / failover-from-fallbacks** (circuit_breaker): circuit-open failover lands on a
   model derived from `router_settings.fallbacks`, never the stale `gpt-4o`.
3. **AC-0**: `0`-budget ⇒ no monitor warn AND no router swap (one test, both layers).
4. **Behavior-change #1**: NO `provider_budget_config` ⇒ no router swap and no monitor warn.
5. **Scoped golden (FIX-5)** (settings): empty/absent config reproduces today's NON-budget
   defaults (session_ttl, smart_thresholds, cost_tiers, warn_ratio) and explicitly does
   NOT preserve provider-budget / failover defaults (now empty).

Plus update existing tests that encoded the now-removed hidden defaults

`TestLoadProviderBudgets` + budget-awareness; `test_fast_circuit_breaker` failover map +
