# 0.5.4-MIGRATE-s3 — design memo

Move the S3 logger onto the canonical `RequestEvent`/recorder seam.
Behavior-preserving: the s3 record is byte-identical when s3 is enabled.
Only the opt-in MECHANISM changes (config-callback-entry → `AIRLOCK_ENABLE_S3_LOGGER`).

## project_s3(event) (projections.py)
Reproduces `AirlockS3Logger._build_record` (s3_logger.py:64-101) field-for-field
from the event, in the exact key order of the old builder:
`timestamp, success, model, user, team, request_id, messages, response, error,
duration_ms, **usage`, then `_redact_record(record)`.
- `response` = `_serialize(event.response_obj) if event.response_obj else None`
- `error`    = `event.bare_exception_error` (s3 emitted `str(exception)|None`, §3.9)
- s3's narrow set has NO guardrail_meta/mcp/served/provider/record_type — not added.

## AirlockS3Logger.record_event(self, event)
`self._append(project_s3(event))` — reuses existing `_append`/buffer/`_flush` path.
DELETE `_build_record` + the 4 callback methods
(`log_success_event`/`async_log_success_event`/`log_failure_event`/`async_log_failure_event`).
KEEP `_append`, `_flush`, `flush()`, buffer+lock, `proxy_s3_logger`, atexit, bucket gate.

## recorder.py
After the enterprise/fathom registrations, gated registration:
```python
if _env_flag("AIRLOCK_ENABLE_S3_LOGGER", default=False):
    from airlock.callbacks.s3_logger import proxy_s3_logger
    request_recorder.register(proxy_s3_logger.record_event, name="s3")  # normal sink
```
NORMAL sink (success+failure, NOT async_only) — s3 historically fired on both sync
and async callbacks. `AIRLOCK_S3_BUCKET` write-gate unchanged.

## Golden equivalence (FROZEN)

## test_s3_logger.py re-point
Tests calling deleted `_build_record`/`log_success_event`/`log_failure_event`/
`async_log_success_event` are re-pointed through `record_event(project_s3(event))`;
assertions preserved (never weakened).
