# 0.5.0-OBS-served — implementation memo

## Goal
Surface the truthful served backend on every response: emit `X-Airlock-Served-By`
(+ `X-Airlock-Served-Region` when present) from the post-call response-headers hook,
attributing the backend that actually served the request (read from the response, not
guessed from the model name). Wire `configure_transparency(config)` into proxy startup.

## Scope (this pack only)
- `airlock/callbacks/model_override_headers.py`: extend
  `async_post_call_response_headers_hook` — call `attribute_served_backend(response)`,
  stash identity into `metadata["airlock_served"]` (best-effort), and merge
  `served_headers(served)` into the returned/additional headers when
  `get_transparency_config().served_headers` is truthy.
- `airlock/proxy.py`: import and call `configure_transparency(config)` at startup
  after `configure_guardrail_overrides(config)`.
- Tests in `tests/test_model_override_headers.py`.

NOT in scope: `X-Airlock-Mutations` / `X-Airlock-Explain` (OBS-headers), enterprise
logger / monitor / mutation sites, `transparency.py` itself (frozen contract), config.yaml.

## Approach
Restructure the hook so served headers are additive and independent of the existing
gemini/override `response_headers`:
1. Compute existing `response_headers` as today (gemini + airlock_response_headers).
2. `served = attribute_served_backend(response)` (no cost_fallback — cost is finalized
   in OBS-log).
3. Stash identity (provider/api_base_host/region/model_id/backend_kind, NOT
   response_cost) into `metadata["airlock_served"]` best-effort, using the metadata
   object via `data.setdefault("metadata", {})`.
4. If `get_transparency_config().served_headers` is truthy, merge `served_headers(served)`
   (which returns `{}` for unknown provider — omitted, never guessed) into
   `response_headers`.
5. If combined `response_headers` is empty → return None (preserves existing
   no-op behavior). Otherwise update `_hidden_params["additional_headers"]` and return
   the dict.

This keeps both streaming (provider via wrapper attribute, handled inside
`attribute_served_backend`) and non-streaming correct, and leaves existing behavior
intact (additive).

## Startup wiring
`proxy.py`: add `from airlock.transparency import configure_transparency` to the
`configure_*` import block and call `configure_transparency(config)` after
`configure_guardrail_overrides(config)`, using the same `config` dict.

## Tests (RED first)
- Non-streaming per backend: bedrock+region, vertex_ai+region, anthropic (no region),
  openai (no region) → correct served headers.
- Streaming-correct: `_hidden_params` lacks `custom_llm_provider`, wrapper attr set →
  header still emitted (CC-T6).
- Unknown provider → no served headers.
- Config opt-out (served_headers false) → no served headers even when provider known;
  reset config to defaults after.
- Existing override / gemini tests still pass (additive).
