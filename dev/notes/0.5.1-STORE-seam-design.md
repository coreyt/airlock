# 0.5.1 STORE-seam — Design memo

Fixes two coupled bugs:

- **R5 undercount** — `ProviderSpend.spend_records = deque(maxlen=1000)` summed over a
  trailing 24h window drops in-window records once a provider does >1000 billed
  calls/day, so daily spend undercounts exactly on the busy providers budgets protect.
- **FIX-1 process-location** — `checkpoint_state`/`restore_state` were wired in
  `proxy.py` (the **launcher**), but spend is mutated only in the **litellm child**
  (`airlock.fast.monitor.log_success_event`). Launcher and child are separate
  processes with separate memory, so the launcher checkpointed an empty store and the
  child's state was never persisted. This silently defeated the breaker checkpoint too.

## Seam interface (`airlock/fast/state.py`)

A new Airlock-owned `SpendStore` backs per-provider spend with **time-bucketed
integer-µ$ counters** in a `DualCache(InMemoryCache(...))`. The DualCache is the
backend so a future Redis swap is a config flip, not a rewrite (its `increment_cache`
maps to Redis `INCR`).

- `SpendStore.record_spend(provider, ts, cost_usd)` — `µ$ = int(round(cost*1e6))`;
  `bucket = int(ts // bucket_width)`; `increment_cache("airlock_spend:{provider}:{bucket}",
  µ$, ttl=TTL)`; track the bucket index in an in-memory index `{provider: set[bucket]}`
  so `recent_spend`/checkpoint can enumerate/prune (DualCache cannot scan keys; a Redis
  backend would use a sorted set instead — same seam).
- `SpendStore.recent_spend(provider, window, now)` — sum the µ$ of buckets whose index
  `>= int((now-window)//bucket_width)`; prune older indices from the index (values also
  expire by TTL). Returns **float USD** (`µ$/1e6`) so `advisor/tools.py:122`, the
  router, and the monitor are unchanged at the call site.
- `ProviderSpend` becomes a thin handle: `record_spend(ts, cost)` / `recent_spend(window)`
  delegate to the StateStore's single `SpendStore`. Public API
  `store.get_provider_spend(provider).record_spend(...)/.recent_spend()` preserved.
- `StateStore._lock` (RLock) is held by `SpendStore` around compound read-modify-write
  (the SpendStore takes the store lock by reference at construction).

### FIX-3 cache config (not DualCache defaults)

`InMemoryCache(max_size_in_memory=100_000, default_ttl=TTL)` and an **explicit per-key
`ttl=TTL` on every set/increment** — never rely on the 600s default. `TTL = window +
bucket_width` (default `86400 + 3600 = 90000s`), strictly ≥ the rolling window.

### Data model

- `bucket_width = 3600s` (1h), `window = 86400s` (24h). Both configurable on `SpendStore`
  so tests can use tiny windows. Accuracy is independent of call volume → R5 fixed.

### Trailing-edge semantic (conservative by design)

Because whole buckets are summed/pruned (`min_bucket = floor((now - window) /
bucket_width)`, keep `index >= min_bucket`), `recent_spend()` and `export_buckets()`
are accurate **to within one bucket width** at the trailing edge and round **UP**: the
bucket straddling `now - window` is counted in full even though part of it predates the
window. This is deliberate and is the **safe** direction for budget protection — budget
warns/swaps fire slightly **early**, never late, so real spend can never silently exceed
a budget by more than one bucket. Excluding the boundary bucket would understate spend
and risk overshooting the budget, which is unsafe, so the conservative inclusion is
kept. A smaller `bucket_width_seconds` tightens the worst-case overage. Pinned by
`test_trailing_edge_is_conservative_within_one_bucket`.

## Checkpoint schema (FIX-7) — sibling file `spend_state.json`

Spend uses a **sibling** file in `AIRLOCK_STATE_DIR`, leaving breaker `cb_state.json`
and its 5-min gate completely untouched (the gate is breaker-only).

```json
{
  "version": 1,
  "timestamp": <epoch>,
  "bucket_width_seconds": 3600,
  "window_seconds": 86400,
  "providers": {"openai": {"<bucket_index>": <micro_int>, ...}, ...}
}
```

- **Versioned** — `version` field; mismatched version → log + skip restore.
- **Atomic** — write `*.tmp.<pid>` in the same dir, then `os.replace` onto target.
- **Prune-before-checkpoint** — `export_buckets()` drops out-of-window buckets before
  writing.
- **Idempotent + replace-not-append restore** — `import_buckets()` uses `set_cache`
  (absolute), not increment, so restoring twice yields the same value.
- **Age-bounded restore** — only buckets within `now-window` are rehydrated; NOT gated
  by the 5-min `_CB_STATE_MAX_AGE_SECONDS` (that stays breaker-only).

## FIX-1 child wiring (`monitor.py` + `proxy.py`)

The monitor module is imported in BOTH the launcher (for `configure_budgets`) and the
litellm child (as the callback). To run restore/checkpoint **only in the child**:

- `proxy.py` sets `env["AIRLOCK_LITELLM_CHILD"] = "1"` on the subprocess env.
- `monitor.py` `_init_child_state()` runs at import **only when that env is "1"**:
  1. **restore-on-startup**: `restore_state(store, cb_path)` (breaker, 5-min gated) +
     `restore_spend(store, spend_path)` (age-bounded).
  2. **periodic checkpoint**: a daemon thread checkpoints both every
     `AIRLOCK_SPEND_CHECKPOINT_INTERVAL` (default 60s).
  3. **shutdown checkpoint**: `atexit` + a SIGTERM handler that checkpoints both then
     chains to the prior handler.
- `proxy.py` **launcher neutralization**: remove the launcher-side `restore_state`
  call (`main()`) and the `checkpoint_state` block in the SIGTERM handler. The launcher
  keeps only the S3 flush. The child is now the single writer of `cb_state.json` (no
  racing second writer) and the sole owner of `spend_state.json`.

The no-network subprocess round-trip test is the durability proof; signal/atexit wiring
is production glue exercised by the smoketest (operator-run).

## Test plan (no network)

`tests/test_fast_spend_store.py`:
1. **R5 accuracy** — >1000 spend events in-window → `recent_spend()` equals true sum;
   contrast that the old `deque(maxlen=1000)` would have dropped records.
2. **Integer µ$** — sub-cent costs accumulate exactly; `recent_spend()` float USD.
3. **TTL ≥ window** — the spend key's `ttl_dict` expiry minus now ≥ window (no 600s).
4. **Rolling window** — out-of-window records excluded from `recent_spend()`.
5. **Checkpoint semantics** — versioned; atomic (temp + `os.replace`); prune-before;
   idempotent (double restore no double-count) + age-bounded (out-of-window not
   rehydrated, in-window is).
6. **Subprocess round-trip** — process A records + checkpoints; a fresh `python -c`
   subprocess restores and asserts in-window present + out-of-window pruned.
7. **Breaker still round-trips** — `cb_state.json` non-CLOSED round-trip on the shared
   (child-located) path; 5-min gate still applies to breaker, NOT to spend.
