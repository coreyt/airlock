# 0.5.3-DECOUPLE — design memo

Goal: enforce one-way layering. `airlock.fast` must NOT import `airlock.guardrails`
(module-level OR lazy). `airlock.guardrails` → `airlock.fast` stays allowed
(`guardrails/observer.py:34` → `airlock.fast.state` is LEFT untouched). Also extract
`airlock/proxy_bootstrap.py` as the only caller of the six `install_*` functions.
Behavior-preserving throughout.

## Relocation map (leaf seams → neutral top-level modules + shims)

1. **text-extraction seam**
   - NEW `airlock/text_extract.py` = the real implementations moved verbatim from
     `guardrails/extract.py`: `extract_text_from_messages`, `_MAX_DEPTH`,
     `_collect_strings`, `extract_text_from_mcp`, `is_mcp_call`, `_BATCH_CALL_TYPES`,
     `is_batch_call`, `extract_text`.
   - `airlock/guardrails/extract.py` → re-export shim:
     `from airlock.text_extract import *` + explicit `__all__` for the 5 public names,
     plus explicit re-export of the private helpers used elsewhere (none, but kept
     importable for compat).

2. **overrides seam**
   - NEW `airlock/guardrail_overrides.py` = real implementations moved verbatim from
     `guardrails/overrides.py`: `_DECISION_KEY`, `_GuardrailOverrideConfig`, `_cfg`,
     `configure_guardrail_overrides`, `_authenticated_client_id`, `_header_value`,
     `resolve_guardrail_decision`. (Lazy `from airlock.admin.tokens import ...` stays —
     admin is not fast/guardrails.)
   - `airlock/guardrails/overrides.py` → re-export shim. MUST expose `_cfg`,
     `configure_guardrail_overrides`, `resolve_guardrail_decision`,
     `_GuardrailOverrideConfig`, `_DECISION_KEY` because `tests/test_guardrail_overrides.py`
     reads/writes `overrides._cfg`. Single source of truth for `_cfg` lives in the
     neutral module (configure/resolve both read the neutral module global).

## Redirect list (the 3 fast → guardrails edges)

- `airlock/fast/monitor.py:27` — `from airlock.guardrails.extract import is_batch_call`
  → `from airlock.text_extract import is_batch_call`
- `airlock/fast/guardian.py:44` — `from airlock.guardrails.extract import extract_text, is_batch_call, is_mcp_call`
  → `from airlock.text_extract import extract_text, is_batch_call, is_mcp_call`
- `airlock/fast/guardian.py:245` (lazy) — `from airlock.guardrails.overrides import resolve_guardrail_decision`
  → `from airlock.guardrail_overrides import resolve_guardrail_decision`

## Bootstrap extraction

- NEW `airlock/proxy_bootstrap.py`: imports the 6 `install_*` functions, exposes
  `bootstrap_airlock_proxy()` that calls them in the EXACT current order from
  `model_override_headers.py:144-151`:
  `docs → circuit_health → error_handlers → admin → batch → models_capability`
  (admin BEFORE batch so the gateway stays outermost — load-bearing).
- `airlock/callbacks/model_override_headers.py`: remove the 6 install imports +
  the module-level call block; add `from airlock.proxy_bootstrap import bootstrap_airlock_proxy`
  and a single `bootstrap_airlock_proxy()` call at module load (same import-time
  trigger via config.yaml:547). Header-hook class + `merge_additional_headers` stay.

## Test designs

- `tests/test_import_layering.py` — dependency-free AST walk of every `airlock/fast/*.py`.
  Collect ALL `import`/`from`-import nodes (incl. function-local/lazy via `ast.walk`).
  Assert none reference a module starting with `airlock.guardrails`. Note: must NOT
  flag `litellm.types.guardrails` (guardian.py:33). Fails if a fast→guardrails import
  is reintroduced (lazy ones included, since `ast.walk` visits nested function bodies).
- `tests/test_proxy_bootstrap.py` — monkeypatch the six `install_*` names in the
  `airlock.proxy_bootstrap` namespace to append their label to a list; call
  `bootstrap_airlock_proxy()`; assert recorded order == the 6-step sequence.
