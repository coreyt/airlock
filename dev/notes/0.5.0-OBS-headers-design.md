# 0.5.0-OBS-headers — implementation memo

## Part A — `X-Airlock-Mutations` response header (must-have, default-on)
Slot: `AirlockModelOverrideHeaders.async_post_call_response_headers_hook`, after the
served-headers block and before `return dict(response_headers)`. Reuse a single
`get_transparency_config()` call (lift the served-headers gate onto the same `cfg`).
Read ledger from `metadata.get("airlock_mutations") or []`; when non-empty and
`cfg.mutation_headers != "off"`, render via `mutations_header(ledger,
cfg.mutation_header_budget_bytes)` and set the header if the rendered value is
non-empty. `mutations_header` already enforces CC-T2 (no content for
non-allowlisted fields) and the byte bound — consume only. Fires streaming +
non-streaming alike; additive (CC-T7).

## Part B — `X-Airlock-Explain` body envelope (opt-in, NON-STREAMING ONLY)
New `async_post_call_success_hook(self, data, user_api_key_dict, response)` on the
same class — non-streaming only by construction (streaming uses the separate
iterator hook, which we do NOT implement → satisfies Decision 7 / CC-T5).

### Body-serialization investigation (empirical)
litellm responses are pydantic `litellm.types.utils.ModelResponse`, whose
`model_config` has `extra="allow"`. Observed:
```
r = ModelResponse(); r.airlock = {"mutations": [...]}
"airlock" in r.model_dump()       -> True
"airlock" in r.model_dump_json()  -> True
```
So a plain attribute assignment (`response.airlock = {...}`) serializes into the
JSON the client receives — no DO NOT TOUCH file needed. Clean attach is viable;
Part B is implemented (not flagged).

### Opt-in + attach
- Read opt-in from `data["proxy_server_request"]["headers"]` (litellm stash),
  compared case-insensitively against `cfg.explain_body_optin_header` (default
  `X-Airlock-Explain`); truthy value (`"1"`, `"true"`, …) → opt-in.
- Absent/falsy → `return response` unchanged (default no-op path, byte-identical,
  CC-T5/CC-T7).
- Opted in → `response.airlock = {"mutations": [asdict(m) for m in ledger]}` where
  ledger is `metadata.get("airlock_mutations") or []` (each `Mutation` via
  `dataclasses.asdict`); then `return response`. Metadata-only, value-safe (CC-T2).
- Guard: only attach when `hasattr(response, "model_dump")` (pydantic) and ledger
  non-empty, so SimpleNamespace/other shapes are left untouched.

Tests assert against the same serialization the proxy uses (`response.model_dump()`).
