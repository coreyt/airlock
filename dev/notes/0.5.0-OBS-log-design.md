# 0.5.0-OBS-log — PLAN memo

**Goal:** Add three additive fields to the JSONL request record produced by
`enterprise_logger._build_record`: `mutations`, `served`, `attribution`.

## Approach
In `_build_record`, just before the `record["airlock_client"] = airlock_client`
assignment (~`:510`), compute and attach:

- `mutations`: `metadata.get("airlock_mutations") or []`, each `Mutation`
  serialized with `dataclasses.asdict` (fall back to `_serialize` for any
  non-dataclass entry). Full ledger, order preserved. Value-safe by construction
  (redact records are value-free; inject `after=None`).
- `served`: `attribute_served_backend(response_obj, cost_fallback=kwargs.get("response_cost"))`
  serialized via `dataclasses.asdict`, or `None` when the helper returns `None`.
  Called here directly (Decision 5) — independent of any header hook.
- `attribution`: `"served"` when `served` is non-None and `served["provider"]`
  is not None; else `"inferred"`.

## Imports
- `import dataclasses` (top of module).
- `from airlock.transparency import attribute_served_backend`.

## Invariants
- `airlock_provider` (inferred) and `record_type == "request"` unchanged.
- No reorder/rename of existing keys; fields are purely additive.

## Tests (RED first) — `tests/test_enterprise_logger.py`
1. Bedrock `_hidden_params` → `served.provider == "bedrock"`, region set,
   `attribution == "served"`, final `response_cost`.
2. No resolvable provider → `served` provider None, `attribution == "inferred"`,
   `airlock_provider` still set.
3. Ledger with a `record_redaction` entry → `record["mutations"]` is the
   serialized full ledger; scrubbed secret value absent; order preserved.
4. Produced when called directly (no header hook) — proves Decision 5.
5. `record_type` still `"request"`; existing fields unchanged.
