# 0.5.4-MIGRATE-sql — design memo

Move the SQL logger onto the canonical `RequestEvent`/recorder seam.
Behavior-preserving: the sql record is field-for-field identical when sql is enabled.
Only the opt-in MECHANISM changes (config-callback-entry → `AIRLOCK_ENABLE_SQL_LOGGER`).

## project_sql(event) (projections.py)
Reproduces `AirlockSQLLogger._build_record` (old sql_logger.py:90-128) field-for-field
from the event, in the exact key order of the old builder:
`timestamp, success, model, user, team, request_id, messages, response, error,
duration_ms, **usage`. NO redaction, NO guardrail_meta/mcp/served/provider/record_type.
- `messages` = `json.dumps(event.messages, default=_serialize)` — a JSON STRING.
- `response` = `json.dumps(_serialize(event.response_obj), default=_serialize) if event.response_obj else None`
  — a JSON STRING (double-encoded) or `None`.
- `error`    = `event.bare_exception_error` (sql emitted `str(exception)|None`, §3.9).

sql stores TEXT columns, so messages/response are JSON strings, NOT objects — the one
shape difference vs `project_s3` (which keeps objects + applies `_redact_record`).

## AirlockSQLLogger.record_event(self, event)
`self._insert(project_sql(event))` — reuses the existing `_insert`/`_ensure_initialized`
path unchanged (engine/table guard + exception swallow). DELETE `_build_record` + the 4
callback methods (`log_success_event`/`async_log_success_event`/`log_failure_event`/
`async_log_failure_event`). KEEP `_insert`, `_ensure_initialized`, `_get_table`, engine/
table, `proxy_sql_logger`, the `AIRLOCK_SQL_URL`-disabled behavior. Dropped now-unused
imports (`datetime`, `json`, `_serialize`); added `from .projections import project_sql`.

## recorder.py
After the s3 gate, gated registration:
```python
if _env_flag("AIRLOCK_ENABLE_SQL_LOGGER", default=False):
    from airlock.callbacks.sql_logger import proxy_sql_logger
    request_recorder.register(proxy_sql_logger.record_event, name="sql")  # normal sink
```
NORMAL sink (success+failure, NOT async_only) — sql historically fired on both sync and
async callbacks. `AIRLOCK_SQL_URL` disabled-path unchanged. Opt-in moves from a
config-callback-entry to `AIRLOCK_ENABLE_SQL_LOGGER` (registered behavior change; DOCS
owns the register entry). Output is identical when enabled.

## Golden equivalence (FROZEN)
`tests/fixtures/0.5.4-sql-golden.json` captured ONCE from the live `_build_record`
(verified `project_sql == _build_record` ignoring only `timestamp`, with messages/
response as JSON STRINGS) BEFORE deletion, then frozen — mirrors the entfathom/s3
oracle-freeze pattern.

## test_sql_logger.py re-point
Tests calling deleted `log_success_event`/`log_failure_event`/`async_log_success_event`
are re-pointed through `record_event(build_request_event(...))`; assertions preserved
(never weakened). `test_async_delegates` → `test_async_dispatch_inserts` (the deleted
async method delegated to the same `_insert`; the normal sink reaches `_insert` on
`recorder.dispatch(is_async=True)`). All sqlalchemy-gated (module skipif unchanged).
