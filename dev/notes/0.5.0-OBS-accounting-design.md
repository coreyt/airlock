# 0.5.0-OBS-accounting — design memo

Pack changes one request-path *number's destination* (CC-T4): which counter spend /
quarantine lands in. Routing unchanged. All new behavior gated on
`get_transparency_config().attribute_accounting_to_served` (default True); flag-off is
byte-identical to today.

## Flag gate
Read the flag once per event via `get_transparency_config().attribute_accounting_to_served`
(imported into `monitor.py`). When False → preserve today's inferred-everywhere behavior.

## SUCCESS path (`log_success_event`, spend) — key off SERVED
After `provider = _infer_provider(model_name)` and `cost = kwargs.get("response_cost", 0.0)`:
when the flag is on, compute
`served = attribute_served_backend(response_obj, cost_fallback=kwargs.get("response_cost"))`
inside a defensive `try/except` (a monitoring callback must never crash the request path;
the test fixture `mock_response_obj` is a bare `MagicMock` whose attribute access makes the
real `attribute_served_backend` raise in `urlsplit`). If `served and served.provider`:
re-key `provider = served.provider` and set `cost = served.response_cost if not None else
kwargs.get("response_cost", 0.0)`. Otherwise leave inferred provider + kwargs cost
(back-compat; never blocks on a served read). `response_obj` is the final served response.
The single `provider` reassignment flows through the existing per-provider counters (spend,
request, success, ratelimit-headroom) — coherent, since on success the served backend *is*
the provider that produced the response and its `_hidden_params` headers.

## FAILURE / 429 path (`log_failure_event`) — key off the ERROR's provider, else inferred
`record_provider_request` / `record_provider_failure` keep the **inferred** provider
(attempt/breaker counting stays per-attempted-provider — Decision 4). Then, before the
`is_rate_limited` gate, when the flag is on, re-key `provider = getattr(exception,
"llm_provider", None) or provider`. The existing `if provider and is_rate_limited:` gate and
the whole quarantine block (record_provider_rate_limit, ratelimit snapshot, metadata, log)
then use that error-provider key. Never reads `served` on failure (often no response).

## Existing tests touched
None weakened. `mock_response_obj` is a MagicMock → `attribute_served_backend` raises →
caught → falls back to inferred, so existing success tests are unaffected. The gemini test
uses a response without `_hidden_params`/`custom_llm_provider` → `served.provider is None` →
inferred `gemini` preserved. Existing 429 tests use `model=gpt-4o` (infers openai) with
`llm_provider="openai"` → same key either way.

## New tests
- success served≠inferred → spend debited to served (`bedrock`), not inferred (`anthropic`).
- success served cost None → amount from `kwargs["response_cost"]`, keyed to served.
- success no served read (`_hidden_params` absent) → inferred + kwargs cost.
- 429 no response (`RateLimitError(llm_provider="openai")`, model infers anthropic,
  `response_obj=None`) → quarantine keyed off `openai`, no crash.
- flag OFF → success + failure both restore inferred behavior.
