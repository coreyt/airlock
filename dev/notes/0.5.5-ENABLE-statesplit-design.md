# 0.5.5-ENABLE-statesplit Design Memo

## Objective

Split `airlock/fast/state.py` (1714 lines) into four focused sub-modules while
keeping `state.py` as a pure re-export facade. Zero logic changes. All 56
existing `from airlock.fast.state import X` callers continue to resolve unchanged.

## Exact Top-Level Names per Sub-Module

### `airlock/fast/_state_core.py`

Classes and functions:
- `BreakerPolicy` (dataclass)
- `_policy_from_mapping` (function)
- `configure_breaker` (function)
- `policy_for` (function)
- `CircuitState` (dataclass)
- `ClientState` (dataclass)
- `SessionRecord` (dataclass)
- `ProviderRateLimitState` (dataclass)
- `ClientProviderState` (dataclass)

- `normalize_client_id`, `NO_CLIENT_ID` (imported from `airlock.client_identity`)

### `airlock/fast/_state_spend.py`

Classes:

Imports needed: `asyncio`, `litellm` (lazy / top-level), constants from

### `airlock/fast/_state_mcp.py`

Classes:
- `McpToolState` (dataclass)
- `McpServerHealth` (dataclass)
- `McpServerState` (dataclass)

Imports needed: `dataclasses`, `datetime`, stdlib only (no circular dep).

### `airlock/fast/_state_persistence.py`

Functions:
- `checkpoint_spend`
- `restore_spend`
- `checkpoint_state`
- `restore_state`

Also imports `tail_jsonl` from ... (tail_jsonl stays in state.py, so
`_state_persistence.py` will define only the functions, and state.py will re-export).

## Import Dependency Graph

```
stdlib
  ↑
_state_mcp.py        (no airlock deps)
_state_spend.py      (no airlock deps, lazy litellm)
  ↑
_state_core.py       ← _state_mcp (TYPE_CHECKING or lazy), _state_spend (TYPE_CHECKING or lazy)
  ↑
_state_persistence.py ← _state_core, _state_spend
  ↑
state.py (facade)    ← all four sub-modules + client_identity + stateprovider seam
```

### Circular Import Mitigation

`StateStore` (in `_state_core.py`) has type references to `SpendStore` and
`McpServerState`. To break the potential cycle:
- Use `TYPE_CHECKING` guard in `_state_core.py`: import `SpendStore` and
  `McpServerState` only inside `if TYPE_CHECKING:` so they are available for
  type hints but not executed at runtime.
- At runtime, the `StateStore.__init__` and methods that actually construct these
  objects will use function-scope imports or rely on the facade having already
  imported all modules.
