# Pack 0.5.3-LATENCY — design memo

Three behavior-preserving latency/concurrency wins (UN-27 + text-cache + vLLM TTL).

## 1. Presidio off the event loop (UN-27)

`AirlockPIIGuard.async_pre_call_hook` runs Presidio's `analyzer.analyze(...)`
synchronously (via `_scrub_mcp_arguments` / `_scrub_messages` →
`_scrub_text_with_mapping`), blocking the asyncio event loop ~50–200 ms/request
and serializing all concurrent PII-enabled requests.

**Approach:** wrap each scrub call in `await asyncio.to_thread(...)`:
- `await asyncio.to_thread(_scrub_mcp_arguments, data, mapping, counters)`
- `data["messages"] = await asyncio.to_thread(_scrub_messages, messages, mapping, counters)`

Only the threading boundary moves. The request-local `mapping`/`counters` dicts
are mutated inside the worker thread under the GIL; ordering is preserved because
the two offloads are awaited sequentially (mcp first, then messages — same order
as today). `record_redaction` / logging stay on the loop. **Redaction output,
mapping, and counters are byte-identical to today** — no change to entities,
thresholds, dedup, or placeholder format.

## 2. Per-request, metadata-scoped, POST-redaction text cache

pii, keyword, and guardian each re-walk the messages via `extract_text`. Make
`extract_text(data, call_type)` cache-aware:
- key `data["metadata"]["_airlock_text"]`; present → return it; else compute from
  the **current** messages/args and store it (via `setdefault("metadata", {})`).
- New request = new `data` = new metadata = fresh cache. No cross-request leak
  (covered by the existing independent-mapping invariant + a new test).

**Order correctness (THE subtlety):** `config.yaml:745+` runs guards in order
`airlock-pii-guard` (first, "redaction — must run first") → `airlock-keyword-guard`
→ … → `airlock-fast-guardian`. pii is the **only** pre_call guard that mutates
`data["messages"]`. So after pii redacts, it calls `refresh_text_cache(data,
call_type)` which **recomputes from the redacted messages and overwrites** the
cache key. Downstream keyword + guardian then read the post-redaction text from
the cache. Extraction runs **once** per request (in pii's refresh). When PII is
disabled, pii returns early without caching; the first reader (keyword) computes +
caches from the unchanged messages, guardian reuses it — still once, still correct.

Staleness guard: only pii mutates messages and it refreshes the cache itself
(`refresh_text_cache` ignores any prior value and overwrites).

Tests pin: (a) with PII enabled the value keyword/guardian get equals the redacted
text (not the original); (b) extraction runs exactly once per request (spy on the
underlying `extract_text_from_messages`).

## 3. vLLM `/models` cache TTL widen + prewarm

`local_vllm_router._DEFAULT_TTL` 5.0 → 30.0 (docstring/env-default mentions too).
Add an idempotent, best-effort `prewarm()` coroutine that calls `_loaded_models()`
(which already swallows httpx errors) so the first real request skips the probe.
Failure is swallowed; a cold cache still resolves correctly on the first request
(lazy `_loaded_models` fetch on miss); non-vLLM aliases never hit the probe.
Wiring: exposed as `router.prewarm()` + module-level `prewarm_local_vllm_router()`
helper, callable from a startup hook; since a clean startup hook is owned by a
DO-NOT-TOUCH file (`proxy_bootstrap.py`), wiring is left lazy (first-request probe)
and noted in output.json rather than forcing a risky edit.

## Test designs

- `tests/test_pii_guard.py`: byte-parity — offloaded hook output == direct
  synchronous `_scrub_messages` output (messages + mapping equal).
- `tests/test_pii_concurrency.py` (new): (a) heartbeat probe — a concurrent
  10 ms asyncio ticker keeps ticking while a slow fake `analyze` (time.sleep)
  runs, proving the loop is not blocked; (b) N concurrent slow PII requests
  finish in ≪ N × single-request latency.
- `tests/test_extract.py`: cache hit returns stored value; cache miss computes +
  stores; post-redaction order (pii refresh → keyword/guardian see redacted text);
  extraction-once spy.
- `tests/test_local_vllm_router.py`: `_DEFAULT_TTL == 30.0` + default `_cache_ttl()`
  is 30.0 + env override; prewarm populates cache; prewarm swallows failure.
