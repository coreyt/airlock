# 0.5.5-IMPL-admission Design Memo

Pack: IMPL-admission — C1 in-loop admission gate

## Module Layout

```
airlock/fast/admission.py        NEW — AdmissionConfig, AdmissionStore, AdmissionGate,
                                        configure_admission, _admission_gate
airlock/fast/settings.py         MOD — AdmissionConfig added; _load_admission; AirlockSettings.admission
airlock/fast/guardian.py         MOD — import _admission_gate; insert Step 2.5 gate block
airlock/proxy.py                 MOD — configure_admission(config) after configure_settings(config)
tests/test_admission.py          NEW — target tests (no-network)
```

## AdmissionStore Pattern

Mirrors SpendStore from `_state_spend.py` but:
- Stores raw integer request counts (not µ$ micro-dollars)
- Uses 1-second bucket width (SpendStore uses 1-hour buckets)
- No DualCache dependency — plain dict of deques (no litellm import required)
- Same threading.RLock discipline

## Startup Wiring

In proxy.py, after `configure_settings(config)`:
```python
from airlock.fast.admission import configure_admission
configure_admission(config)
```

`configure_admission` reads config via `load_airlock_settings`, checks `settings.admission.enabled`,
and sets the module-level `_admission_gate`. Off-by-default: gate is None unless explicitly enabled.

## Guardian Insertion Point

After Step 2 (threat assessment, lines 275-282 in guardian.py), before the
`if not mcp and not batch:` block (line 286). The gate is unconditional (applies to
all call types including MCP and batch). Inserts as Step 2.5.

Uses `_priority_for_gate` local variable (NOT reusing Step 4's `priority`) to avoid
naming conflicts. The Step 4 `compute_priority` call is preserved unchanged.

## Header Wiring Decision

`data["metadata"]["airlock_admission"]` is stamped in the guardian hook.
Checked `airlock/transparency.py` — it is NOT in the MODIFY list. Investigation:
transparency.py reads from `metadata["airlock_response_headers"]` to set headers.
To wire `X-Airlock-Admission` as a response header, a follow-up pack would add
`metadata["airlock_response_headers"]["X-Airlock-Admission"]` to the guardian code,
or extend transparency.py. Stamping metadata is sufficient for this pack (d-028
follow-up).

## Fail-Open Invariant

Any exception in AdmissionGate.check() → logs warning → returns (True, 0.0).
Never blocks a request because the limiter errored.

## Off-by-Default

`_admission_gate is None` at module import. The gate block in guardian is a single
cheap None-check. No performance impact when disabled.

## Concurrency Check

`sem._value` peek is non-blocking (no await). This is synchronous — check() has
no async code. The semaphore is acquire()d by the caller at the async layer
(future work); for now the peek enforces the concurrency cap without acquiring.

NOTE: The concurrency cap peek via `sem._value` is a best-effort non-blocking
check. Actual acquire/release wiring (to prevent double-counting) is a follow-up.

## Metrics

No metrics emitted in this pack. AdmissionStore is keyed by client_id only.
Does not read ModelState or interact with the circuit breaker.
