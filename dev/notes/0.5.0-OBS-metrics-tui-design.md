# 0.5.0-OBS-metrics-tui — design memo

Observe-only pack. Two additive surfaces, no contract changes beyond one helper.

## Part A — `airlock_mutations_total{field,op}`
- New Counter in `_build_metrics` (`airlock/callbacks/metrics.py`), labels `field`/`op`
  (both bounded: `op` is the 7-value MutationOp enum; `field` is a small fixed set of
  request keys), so label cardinality is safe.
- `_record_mutations(metadata)` iterates `metadata.get("airlock_mutations") or []` and
  increments per `Mutation` (`.field`/`.op`). Guards a non-iterable/empty ledger and
  skips records missing field/op — never crashes.
- Called from `AirlockMetricsCallback.log_success_event`. The ledger is only populated
  pre-call, so success is the right (single) place — no double-count risk with the
  failure path (CC-T1 one ledger).
- `metrics_callback` module instance + `_self_register()` mirroring
  `enterprise_logger.py` / `monitor.py`. LiteLLM's `logging_callback_manager` dedupes,
  so repeat calls are idempotent.

## Part B — `classify_backend_kind` + TUI "Served via"
- Extracted the provider→backend_kind logic from `attribute_served_backend` into a
  public module-level `classify_backend_kind(provider)` in `transparency.py`;
  `attribute_served_backend` now calls it (behavior preserved — existing tests green).
  This is the ONLY transparency.py surface change (purely additive).
- TUI overview: appended a "Served via" column to the providers table
  (`add_columns`, row tuple, empty fallback row). Cell = `classify_backend_kind(name)`
  where `name` is the provider from `store.all_providers()`. No store/per-request state.

## Out of scope (untouched)
enterprise_logger, monitor, model_override_headers, breaker state machine, config.yaml,
pyproject/uv.lock, transparency dataclass fields / existing signatures.
