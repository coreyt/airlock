# 0.5.2-CAP-v1models — design memo

## Goal
Additive ASGI response-transform seam that folds an `airlock` sub-object
(`capability_record(entry)`) into each model on `GET /v1/models` and `GET /models`.
Purely additive — never remove/rename a standard OpenAI field.

## Components (`airlock/models_seam.py`)
1. `_build_capability_map()` — load config the same way `model_alias.load_from_config`
   does (`AIRLOCK_CONFIG` env → default `config.yaml`; tolerate missing/invalid → empty
   map, warn). Returns `{model_name: capability_record(entry)}`. Reuses
   `capability.capability_record` (NOT reimplemented).
2. `ModelsCapabilityMiddleware(app, capability_map)` — pure ASGI class. Pass through
   untouched unless `type==http`, `method==GET`, `path in {/v1/models, /models}`.
   For the targeted paths: buffer the response, and ONLY transform when status==200 AND
   content-type is JSON AND body parses to a dict with a list `data`. Augment each
   `data[i]` that is a dict with an `id` in the map by setting `item["airlock"]`.
   Re-serialize, fix `content-length`, drop `transfer-encoding`. On ANY error → emit the
   original bytes unchanged (debug log). Stream-safe: buffer until `more_body` false.
3. `install_models_capability_seam_on_proxy_app() -> bool` — mirrors
   `batch/middleware.py:546-583` dual pre/post-start install + idempotency flag
   (`app.state.airlock_models_seam_installed`).

## Wire-in
One call line in `callbacks/model_override_headers.py` after
`install_batch_gateway_on_proxy_app()` (:151), plus a top-level import.
