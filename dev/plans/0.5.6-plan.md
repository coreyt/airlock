# Airlock 0.5.6 — Plan

Release theme: **model catalog correctness + GPT-5.6**. Two things drove it: the
shipped CLI template targeted model ids that are retired or do not exist (and the test
that should have caught it only ever checked one of the two configs), and OpenAI's
GPT-5.6 family needed adding without the silent-cost-substitution traps its
sol/terra/luna naming invites.

**Status: ✅ RELEASED 2026-07-21.** Tagged `v0.5.6` at `8159116`, published to PyPI
(`airlock_llm-0.5.6` wheel + sdist), GitHub release live. Verified by clean install
from PyPI: `__version__ == 0.5.6`, litellm resolves to 1.93.0. CI green on all four
jobs at the tagged commit. Full suite **2695 passed**. Shipped two days ahead of the
2026-07-23 `gpt-5.1-codex` retirement.

**Scope expanded 2026-07-20:** with everything landing the same day, the "urgent patch
must stay minimal" split bought nothing, so 0.5.6 now carries the whole model-catalog
change. See H-3..H-7.

**Not in scope:** admission gate hardening (0.5.7); ENFORCEMENT of strict validation
(0.5.8 — gated on the warn-only measurement window, which cannot be compressed).

**Numbering:** this hotfix takes `0.5.6`; the admission-gate plan moved to `0.5.7` and
the GPT-5.6 work to `0.5.8`. (`0.5.5.1` was proposed and withdrawn — not valid semver.)

## H-1 — Template ships broken and retiring model ids

**Severity: urgent.** Every user running `airlock init` gets a config with dead targets.

| alias | was | problem | now |
|---|---|---|---|
| `gpt-5-codex` | `openai/gpt-5.1-codex` | **retires 2026-07-23** | `openai/gpt-5.3-codex` |
| `gemini-pro` | `gemini/gemini-3.1-pro` | **does not exist** — only `-preview` | `gemini/gemini-2.5-pro` |
| `gemini-flash` | `gemini/gemini-3-flash` | **does not exist** — only `-preview` | `gemini/gemini-2.5-flash` |
| `claude-opus` | `anthropic/claude-opus-4-6` | two generations stale | `anthropic/claude-opus-4-8` |
| `gpt-5-pro` | `openai/gpt-5.4-pro` | a generation behind root | `openai/gpt-5.5-pro` |
| `gpt-5` | `openai/gpt-5.4` | a generation behind root | `openai/gpt-5.5` |
| `gpt-5.4` | *(missing)* | root's fallback chain references it | added |

The two Gemini entries were **already broken** — not future-dated like the codex
retirement. Verified against live `GET /v1/models` on both providers (2026-07-20), not
inferred from litellm's map.

Separately, `gemini-flash-lite` was advanced to `gemini/gemini-3.1-flash-lite` in
**both** template and root — 3.1-flash-lite is GA, unlike 3.1-pro / 3-flash which are
preview-only.

## H-2 — The test gap that allowed the drift

`tests/test_config_consistency.py` validated fallbacks against the **template fixture
only**. `root_config` / `root_model_names` fixtures existed but were unused, so drift
in either direction went uncaught.

**Changes:**
- Parametrized `any_config` fixture — every consistency check now runs against **both**
  the template and the root config.
- `TestNoRetiredModels` + `RETIRED_MODEL_IDS`, seeded with the 2026-07-23 codex cohort
  (`gpt-5-codex`, `gpt-5.1-codex`, `gpt-5.1-codex-max`, `gpt-5.1-codex-mini`,
  `gpt-5.2-codex`). Strips the provider prefix before comparing.
- `TestTemplateRootParity` — template and root must agree on every alias they share.
  This is what caught the Gemini and `claude-opus` drift.

## H-3 — `_apply_cost_tier` force-swap on alias miss

`_apply_cost_tier` is a plain membership test that **force-swaps to `tier_models[0]`
on a miss** — so an alias absent from its tier is not left alone, it is silently
rerouted to a different model. Adding 8 callable GPT-5.6 aliases made this live.

- `airlock/fast/router.py`: added `_alias_body_map` (built in `set_router_config`); an
  alias sharing its `litellm_params.model` body with an in-tier model is now recognised
  as in-tier. Genuine out-of-tier swaps still happen — that is the feature.
- Every callable 5.6 alias is listed in exactly one tier, in `config.yaml` **and** both
  duplicated `_DEFAULT_COST_TIERS` dicts (`router.py`, `settings.py`), verified identical.

## H-4 — GPT-5.6 model entries

Root + template: `gpt-5.6-sol` / `-terra` / `-luna`, bare `gpt-5.6`, and four
`openai/`-prefixed twins. Order is load-bearing (`-sol` precedes bare `gpt-5.6` so dated
snapshots resolve to the explicit variant). Fallback chains for all eight, grouped by
body; Luna deliberately does **not** fall back to `claude-haiku` (200K cap vs 1.05M
would hard-fail rather than degrade).

**Bare `gpt-5.6` is NOT a real OpenAI id** — verified live against `/v1/models`, which
offers only the three variants. The alias is pinned to the explicit sol body so that
string is never forwarded upstream.

## H-5 — Dropped-qualifier guard (the GPT-5.6 cost trap)

`gpt-5.6-mini` / `-nano` / `-pro` do not exist but are the obvious guesses, and scored
0.57 against bare `gpt-5.6` — **Sol, the most expensive variant** — routing there
silently at DEBUG. A caller reaching for the cheap model was billed 5x and never told.

`airlock/fast/model_alias.py` now refuses any fuzzy match that would **ignore a
non-numeric token the caller supplied**: qualifiers are how a catalog distinguishes
price tiers, so dropping one serves a different model than was asked for. Numeric
tokens remain version noise, so `gpt-5.6-sol-2026-07-09` still resolves.
`suggest()` returns ranked candidates with tier for a "did you mean".

> A first attempt used a cost-tier ambiguity heuristic. It **did not fire** — the close
> candidates were all high-tier — and was replaced. Recorded so it is not retried.

## H-6 — litellm floor `>=1.93.0`

Rationale is **determinism, not capability**. Verified with
`LITELLM_LOCAL_MODEL_COST_MAP=True`:

| litellm | `gpt-5.6-sol` |
|---|---|
| 1.89.0 | **ABSENT → $0.00** |
| 1.93.0 | $5.00/$30.00 per 1M |

Silent zero-cost accounting on a $5/$30 model makes budgets, provider spend, and
near-limit routing all under-count.

**Verification caveat:** the litellm-facing suite (546 tests) passes identically on
1.89.0 and 1.93.0 in an isolated venv. The **full** suite was not run on 1.93.0 —
that venv lacks the `en_core_web_lg` spacy model and the `db` extra, causing 78
unrelated errors. The main `.venv` was deliberately NOT upgraded: **the live service
runs from it** (PID 2933806).

## H-7 — Warn-only `reasoning_effort` detection (back-ported from 0.5.8)

Additive and **routing-identical** — no value sent upstream changes. Emits a WARNING
`event=effort_would_reject` plus an advisory ledger entry so the cohort is countable
via the normal observability path. Shipping it here starts the measurement clock
earlier; enforcement stays in 0.5.8.

**Semantics corrected during back-port:** the check now evaluates the value the
**client sent**, not the value Airlock emits after translation. Enforcement validates
the client's value, so `none` on `gpt-5.4` will 400 — but today's translation to
`minimal` is supported, so measuring the emitted value reported that cohort as fine
and hid the single largest group the breaking change hits.

`max` is treated as unsupported (litellm sets no `supports_max_reasoning_effort` flag
and no funded key was available to test it) — marked in-code for revisit.
