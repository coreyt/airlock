## Verdict: BLOCK

### 1. [high] Batch gateway bypasses proxy auth
Refs: `airlock/batch/middleware.py:24-29`, dispatch path.
The ASGI middleware dispatches `?custom_llm_provider=aistudio` requests before
LiteLLM's route-level auth dependency, and `airlock/batch/` has NO
`Authorization`/`AIRLOCK_MASTER_KEY` check. The new `/v1/files` + `/v1/batches`
ingress is unauthenticated. **Orchestrator-verified: real.** Fix: enforce the
master key in the gateway before any upload/create/cancel (401 otherwise).

### 2. [high] Expired-lease reclaim not race-free (§3.7)
Refs: `airlock/batch/store.py:84,103`, `gateway.py:188-214`.
An expired `CREATING` loser doesn't re-acquire the lease under CAS before
`list_jobs()` + provider create; multiple reclaimers can all create → violates the
≤1-job bound. Fix: CAS-reacquire/extend the lease on reclaim.

### 3. [high] RETRIEVING gate allows concurrent full result fetches (§3.7)
Refs: `airlock/batch/store.py:153-171`, `gateway.py:238-249`.
`begin_retrieving()` returns True when already RETRIEVING → concurrent pollers both
fetch/process the whole result file. Fix: only the CAS winner fetches; others no-op.

### 4. [medium] Create path re-buffers the whole upload
Refs: `airlock/batch/runtime.py:68`, `gateway.py:221-232`, `aistudio.py:217`.
Create reads the stored JSONL into one `bytes`, translates to a list, rejoins, and
uploads via `BytesIO` — unsafe at the stated ~2GB. Fix: stream/line-iterate.

### 5. [medium] Tests miss the load-bearing security/concurrency cases
Refs: `tests/test_batch_gateway.py:253,290,382`.
No concurrent expired-lease reclaim, concurrent RETRIEVING, or gateway-auth test;
the sync-marker test validates a helper, not an end-to-end provider call.
