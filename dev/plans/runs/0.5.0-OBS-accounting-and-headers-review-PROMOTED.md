## OBS-accounting — Verdict: CONCERN → FIXED
**Target:** `obs-accounting` impl `7e86758`; fix `999ec01`; merged `b079b97`.

### 1. [medium] Served-attribution failure silently falls back to inferred billing — FIXED

raise and asserts the WARNING + correct inferred-provider spend. 33 tests green.

## OBS-headers — Verdict: CONCERN → FIXED
**Target:** `obs-headers` impl `3e2cd22`; fix `b68f626`; merged `13110c2`.

### 1. [medium] Header values not CR/LF-normalized — FIXED
`mutations_header`/`served_headers` interpolated `field`/allowlisted `after`/`provider`/
`region` directly; a CR/LF in an allowlisted value (e.g. a model name) could inject a
header. **Fix:** `_header_safe()` strips `\r`/`\n`, applied at every value interpolated into
a header string in both serializers (no allowlist/byte-bound/signature change). 4 new tests;
43 green. **This also closes the OBS-served deferred CRLF nit** (served_headers now sanitized).

**Codex passed:** default response body byte-identical without `X-Airlock-Explain` opt-in
(CC-T5/CC-T7); Part B guarded to `model_dump` objects, streaming excluded; OBS-served block
intact; `mutations_header` allowlist/byte-bound unchanged; scope = model_override_headers.py
(+ the authorized transparency.py header-safety hardening).
