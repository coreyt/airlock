### 1. [medium] `guardrails.overrides` shim exposed stale `_cfg`
Refs: `airlock/guardrails/overrides.py:17`, `airlock/guardrail_overrides.py:54,130`,
`tests/test_guardrail_overrides.py:21`

The shim imported `_cfg` by value, but `configure_guardrail_overrides()` rebinds
`_cfg` in the canonical `airlock.guardrail_overrides`. So `airlock.guardrails.overrides._cfg`
went stale, and the `_env` fixture's `saved = overrides._cfg` / `overrides._cfg = saved`
read a stale object and wrote a dead shim attribute (the canonical global was never
restored → potential config-state leak between tests). No production impact (prod uses
the `configure_*` function, which correctly rebinds the canonical global), but not true
shim parity for the module-state surface.

CONCERN, no BLOCK. I fixed it directly in the worktree rather than overriding:

- **Shim → PEP 562 read-proxy.** `airlock/guardrails/overrides.py` no longer imports

  live state. Public `configure_guardrail_overrides` / `resolve_guardrail_decision`
  stay as explicit re-exports.
- **Fixture → canonical save/restore.** `tests/test_guardrail_overrides.py` now
  snapshots/restores `airlock.guardrail_overrides._cfg` (the single source of truth),
  so the restore is effective and no config state leaks.
