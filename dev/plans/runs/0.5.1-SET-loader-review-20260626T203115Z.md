## Verdict: CONCERN

### 1. [medium] Unconfigured defaults omit budget windows
Refs: `airlock/fast/settings.py:97,100,364,367`, `tests/test_fast_settings.py:348`

`load_airlock_settings({})` builds default `budget_windows`, but `AirlockSettings()`
defaults `budget_windows` to `{}`. Since `get_settings()` returns `AirlockSettings()`
when unconfigured, that accessor does not return consistent safe defaults for every
field (provider_budgets present, windows empty). Test only checks `session_ttl`.

### 2. [medium] Failover env shape only partially validated
Refs: `airlock/fast/settings.py:196,201`, `tests/test_fast_settings.py:198,209`

`AIRLOCK_FAILOVER_MAP` accepts any dict and iterates each value; a malformed value like
`{"a": "bc"}` becomes `{"a": ["b","c"]}` (string iterated char-by-char) instead of
warning and falling back. Violates the `dict[str, list[str]]` typed boundary and the
DoD's malformed-input-fallback requirement. Tests cover bad JSON but not wrong inner shapes.

## What passed on inspection
Pack is additive — `proxy.py` constructs settings without changing consumers; existing
defaults preserved; budget parsing preserves `0`, handles `budget_limit`/`max_budget`,
uses LiteLLM `duration_in_seconds` for `"1d"`; precedence-matrix tests cover the main
env > config > default paths + fallback list conversion + singleton wiring.

## Orchestrator triage — FIX both (no override)

Both findings are legitimate medium robustness issues at the typed config/env boundary
and are cheap to fix; neither is a BLOCK. Decision: **FIX** (not override).
- Finding 1: make the unconfigured accessor return consistent defaults — `get_settings()`
  returns `load_airlock_settings({})` (or align the `AirlockSettings()` window default to
  the budget default), so every field is internally consistent. Add a test asserting
  `get_settings()` (unconfigured) has windows matching its budgets. (Note: SET-unify will
  empty the budget defaults per Q1, which also moots the inconsistency — but fixing it now
  keeps SET-loader self-consistent.)
- Finding 2: validate each `AIRLOCK_FAILOVER_MAP` value is a `list` before coercing;
  warn-and-fallback on a non-list inner value. Add a wrong-inner-shape test.
