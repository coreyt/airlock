## Verdict: NEEDS_FIXES (fallback) → FIXED → PASS-after-fix

### 1. [critical] Null request metadata crashes the hook hot-path — FIXED
Refs: `airlock/callbacks/model_override_headers.py` (served stash, ~`:56-65`)

The `airlock_served` stash used `data.setdefault("metadata", {})`, which returns the
existing value when the key is present. A client sending `"metadata": null` left
`data["metadata"] = None`, so `setdefault` returned `None` and
`metadata["airlock_served"] = {...}` raised `TypeError` — a 500 on every successful
call (resolvable provider) whose request nulled metadata. Untested path.

**Fix (`f6f0fae`, RED `0c3ab3a`):** explicit `isinstance` guard — when
`data["metadata"]` is missing/None/non-dict, write a fresh `{}` back into `data` and
stash into a separate `served_meta` local (the `metadata` local used by the gemini
path is untouched). Two regression tests added (`"metadata": null` and missing-key),
**14/14 green**. The five identity fields and all header-emission logic below the
stash are unchanged. Orchestrator-verified by reading the diff.

### 2. [nit] No CRLF stripping on served header values — DEFERRED (follow-up)
Refs: `airlock/transparency.py:259-261` (`served_headers`)

`provider`/`region` are written into header values without CRLF stripping. These come
from LiteLLM's internal routing tables (operator config, not live user input), so
exploitability is low. **Deferred:** `served_headers` lives in `airlock/transparency.py`
(OBS-core's frozen module, out of OBS-served's scope). Tracked as a defense-in-depth
follow-up to fold into OBS-headers (which also serializes header values) or a small
OBS-core touch — `str(v).replace("\r","").replace("\n","")` on emitted header values.
