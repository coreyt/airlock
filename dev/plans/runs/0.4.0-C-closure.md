## Review arc
1. **C (initial)** — codex BLOCK: 3 high + 2 medium (auth bypass; expired-lease
   reclaim race; RETRIEVING concurrent fetch; ~2GB re-buffer; test gaps).
   Verdict: `0.4.0-C-review-*.md`.
2. **fix-1** (`e27be4e`) — codex re-review (`0.4.0-C-fix1-review-20260615T130105Z.md`):
   - **#1 Auth — CLOSED.** Master-key check in the gateway before any handling
     (`hmac.compare_digest`; 401 when set; open when unset for proxy parity).
   - **#4 Streaming — CLOSED.** Create streams `input_path` line-by-line to a temp
     file; `upload(src: str)` streams from disk; temp cleanup on error.
   - **#2/#3 concurrency — PARTIALLY** (immediate CAS fixed; lease-expiry fencing
     remained). codex held the impl to **exactly-once**.
3. **HITL decision (2026-06-15):** design §3.7 deliberately targets *at-least-once
   with duplicates bounded to ≤1 and auto-cancelled* (exactly-once is impossible:
   non-idempotent provider create, no provider idempotency key). The code
   implements that bound (`gateway.py:208-213` reconcile→`list_jobs(idem)`→
   `cancel(extra)`→`batch_duplicate_cancelled`; per-row `ON CONFLICT` upsert).
   **Accept the design bound + prove it with tests** rather than chase exactly-once.
4. **fix-2** (`97b7c6e`, test-only) — PROVES the bound holds:
   - Duplicate create under an EXPIRED lease → reclaimer CAS-reacquires, adopts the
     earliest orphan, cancels the extra → **exactly one surviving job**; no-orphan
     case creates exactly one.
   - Slow/repeated fetch under an expired RETRIEVING lease → **idempotent per-row
     staging** (each row staged once, content_sha stable, no duplicate output).
   All proofs pass; no production change; had any proof failed it would have been a
   real defect to escalate.

## Verdict: CLOSED
Security finding (auth) and the resource finding (streaming) are fixed and
codex-confirmed. The residual concurrency findings are the **accepted §3.7
at-least-once bound** (HITL-authorized) and are now **demonstrated by tests**, not
left as a claim. This is a documented, HITL-authorized acceptance of an
exactly-once-impossible tradeoff — not a silent BLOCK override.
