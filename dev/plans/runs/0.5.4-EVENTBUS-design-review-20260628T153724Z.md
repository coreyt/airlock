## Verdict: BLOCK

### 1. [high] `RequestEvent` cannot reproduce s3/sql failure `error`
Refs: design-request-event-bus.md:116, s3_logger.py:93, sql_logger.py:121, enterprise_logger.py:73

§4 only puts `error/error_type/failure_category` on the event. s3/sql emit the **exact bare** `str(kwargs.get("exception"))` (incl. `None`/empty cases) while enterprise normalizes. The event must carry the **raw bare exception string** as its own field so s3/sql projections reproduce it; the rich triple alone can't.

### 2. [high] Fathom optional fields missing from the event superset
Refs: design-request-event-bus.md:101, fathom_logger.py:154/158/170

Fathom can emit `response_text`, `headers_json`, `mcp_arguments_json` behind env flags, sourced from `kwargs`/`response_obj`. The event has `response`/`response_cost`/narrow MCP meta but **not** request headers, MCP arguments, or a raw response-text source — so a pure `project_fathom(event)` can't be field-for-field equivalent without re-reading kwargs (violates the projection seam).

### 3. [medium] Inventory omits request-shaped producers outside `_build_record`
Refs: design-request-event-bus.md:27, enterprise_logger.py:236 (`write_precall_block_record()`), enterprise_logger.py:284 (`write_batch_record()`)

The "3 builders + fathom + 2 side channels" inventory is right for `_build_record()` defs but misses two other request/log-record producers. The note must **migrate them or explicitly scope them out** (today only `admin_action` is excluded).

### 4. [medium] One-callback seam under-specifies self-registration + monitor ordering
Refs: design-request-event-bus.md:134, config.yaml:542, enterprise_logger.py:572, metrics.py:203, fast/monitor.py:320

The design doesn't say how existing module-level self-registration (enterprise, metrics, fast monitor) + config.yaml registration are disabled/replaced by the one fan-out callback. Fast monitor mutates `metadata["airlock_provider_protection"]` and is **order-sensitive** — sourcing the event before vs after it changes enterprise `guardrail_meta`.

## What passed on inspection
Core `_build_record()` inventory correct (enterprise/s3/sql + fathom reuse); per-sink redaction, sql JSON-string encoding, usage/response_cost, timestamp-convergence-as-registered-change, and Gemini "enrich once before fan-out" all correct; metrics scope call correct (only requests_total/request_duration/mutation counting at the seam); served attribution flows through the ACL.
