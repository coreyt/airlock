## Verdict: CONCERN

### 1. [medium] Tiny budgets can exceed the byte bound
Refs: `airlock/transparency.py:241` (and the suffix return ~`:252`)

`mutations_header` measures UTF-8 bytes for normal truncation, but if even the bare
`…+N more` suffix is larger than `budget_bytes`, it returns that suffix anyway. Since
the config accepts any positive budget, `budget_bytes=1` would emit an over-budget
header — violating the "true UTF-8 byte bound" contract.

### 2. [low] Boolean config coercion is surprising for string values
Refs: `airlock/transparency.py:314`

`served_headers=bool(served)` / `attribute_accounting_to_served=bool(accounting)`
make quoted YAML strings like `"false"`, `"0"`, `"off"` evaluate to `True`. Defaults
are safe (real YAML booleans parse correctly), but string-valued config does not meet
"booleans coerced sanely".

## What passed on inspection
CC-T2 header safety solid (inject/rewrite/drop render `field=<op>` with no value;
redact values rejected in `Mutation.__post_init__`). `attribute_served_backend` has
the streaming provider fallback, cost fallback, backend-kind mapping, and partial
`provider=None` handling. `detect_dropped_params` is scoped to recognized OpenAI
params and handles `None` lookups safely. Config defaults + `get_transparency_config()`
before `configure_transparency` correct. **Zero call-sites changed** confirmed.

## Orchestrator triage

**Decision: FIX both, then re-verify (do NOT override).** Both findings are real and
sit in the foundational module every later pack imports, so correctness here compounds.
Neither touches the public dataclass/helper *contract* (signatures unchanged), so
downstream prompt authoring is unblocked by the fix.

- **F1 (medium):** when the `…+N more` suffix alone exceeds `budget_bytes`,
  `mutations_header` returns `""` (omit the header) rather than emitting an
  over-budget string. Add a tiny-budget test asserting the result never exceeds the
  byte bound (and is `""` when nothing fits).
- **F2 (low):** coerce string booleans sanely in the config loader — treat
  `"false"/"0"/"off"/"no"/""` (case-insensitive) as `False` and `"true"/"1"/"on"/"yes"`
  as `True` for `served_headers` and `attribute_accounting_to_served`. Add a
  `served_headers: "false"` → `False` test.
