# Airlock 0.5.3 — Plan

Release theme: **decouple from LiteLLM internals and unblock the hot path — an
Anti-Corruption Layer, a Presidio that doesn't stall the event loop, and the
structural hygiene that keeps `fast/` and `guardrails/` honest.** A point release
of *correctness, latency, and coupling* fixes surfaced by the 2026-06 architecture
audit (`dev/notes/architecture-audit-0.5.0-2026-06.md`). No new user-facing
features; every change is behavior-preserving except the documented latency win.

**In scope (Tier 1 latency + Tier 2 structural from the audit):**
1. **PII off the event loop** — the only verified hot-path latency hazard.
2. **LiteLLM Anti-Corruption Layer (ACL)** — one module owns every read of
   LiteLLM internals (`_hidden_params`, wrapper attributes, the proxy-app object).
3. **Break the `fast`↔`guardrails` import cycle** + extract a dedicated
   `proxy_bootstrap.py` from `model_override_headers.py`.
4. **Shared request-text extraction** + **`threat_score` race fix** (small
   correctness/latency cleanups in the guard chain).

**Deferred (deliberate scope choices, not constraints — see "Deferred"):** the
observability event-bus unification (audit Tier 3 #8) is a **candidate/stretch**
here and drops out cleanly if the release is getting large; the `state.py` split
and the `StateProvider` de-globalization (Tier 3 #7/#9) move to **0.5.4** because
they pair with the multi-worker/isolation work there.

> Motivating audit findings (2026-06-25): (a) `pii_guard.py:212` calls Presidio's
> synchronous `analyzer.analyze` inline inside an `async def`, blocking the event
> loop ~50–200ms/request whenever `AIRLOCK_PII_ENABLED` — serializing concurrency
> under load. (b) Reads of LiteLLM internals (`response._hidden_params`,
> `response.custom_llm_provider`, `sys.modules.get("litellm.proxy.proxy_server")`
> poking `app.middleware_stack`) are scattered across `transparency.py`,
> `monitor.py`, `model_override_headers.py`, `batch/middleware.py`, `admin/http.py`
> against a wide version range (`>=1.83.4,<2`) but 1.89.0-pinned behavior. (c) A
> bidirectional package dependency exists: `fast/guardian.py:41` imports
> `guardrails.extract` while `guardrails/observer.py:34` imports `fast.state`.

Orchestrated with the agent harness — see `dev/agent-harness-runbook.md`,
`dev/agent-harness-reference.md`, and this directory's `README.md`. Parent train:

behavior-preserving — so the harness's role here is the **parity oracle**, not new

  `dev/smoketest/`**: capture `served_header_client.py` output (served-by, region,

  end-to-end witness that the unit parity tests are complete.

  confirmation (PII redaction unchanged **and** concurrent requests no longer

  changes any logged/emitted client-visible field, that pack **extends**
  `dev/smoketest/` accordingly; otherwise no harness change is required.

### 0.5.3-ACL — scope guardrails

- **Read-only inventory first.** Enumerate every `_hidden_params`,
  `.custom_llm_provider`, `response_cost`, `additional_headers`, and
  `sys.modules.get("litellm.proxy.proxy_server")` access in the tree; the ACL must
  cover all of them or it is not an ACL. List the inventory in the pack's
  `output.json`.
- **No behavior change.** The ACL is a pure extraction: same values, same
  streaming-vs-non-streaming branches (CC-T6), same "unknown provider ⇒ omit the
  header, don't guess" rule (`dev/architecture.md` §3.7). Tests assert byte-for-byte
  parity on the header/attribution outputs before vs after.
- **Single home for version assumptions.** Move the "LiteLLM 1.89.0
  `_hidden_params` contract" note into the ACL module docstring; downstream code
  stops embedding version assumptions. When LiteLLM bumps, this is the **one** file
  to re-verify.
- **Do not widen scope to a full provider-adapter registry** — that is a larger
  refactor; the ACL only isolates *reads of internals*, not provider dispatch.

### 0.5.3-LATENCY — scope guardrails

- **Behavior-preserving offload.** Presidio results must be identical; only the
  threading changes. Add a concurrency test: N simultaneous PII-enabled requests
  no longer serialize (assert wall-clock ≪ N × single-request latency, or assert
  the event loop is not blocked via an interleaving probe).
- **Text-extraction cache is per-request, metadata-scoped**, computed on first use
  and invalidated implicitly (new request = new `data`). Guard against a guard
  mutating messages *after* extraction (PII redaction does mutate — the cache must
  live downstream of PII or be recomputed post-redaction; pin the order in the
  test). PII redaction order vs the cache is the one correctness subtlety —
  document it.
- **vLLM TTL + prewarm** is opt-in safe: a cold cache must still resolve correctly
  on first request (no regression for non-vLLM aliases, which never hit this path).

### 0.5.3-DECOUPLE — scope guardrails

- **Dependencies point one way after this pack.** Verify with a quick import-graph
  check (`grep`/`pydeps`): no module under `airlock/fast/` imports
  `airlock/guardrails/` and vice-versa, OR the shared seam lives in a neutral
  module both may depend on. Add a regression guard (an import-linter contract or a
  test that fails on a re-introduced cycle).
- **`proxy_bootstrap.py` is the only place that calls `install_*`.**
  `model_override_headers.py` shrinks to the response-header hook alone. The
  bootstrap's order (batch gateway → admin perimeter → header middleware →
  LiteLLM) is preserved exactly (it is load-bearing — see `dev/architecture.md`
  §3.6 mount order); assert the order in a test.

### 0.5.3-RACE — scope guardrails

- **The race fix is the priority**; identity/config consolidation are bundled only
  because they touch the same files. If consolidation balloons, split it out and
  ship the race fix alone.
- **No semantic change to client identity** — the consolidation must produce the
  *same* `client_id` for every existing input (golden test over the 3 old paths'
  outputs).

## Behavior-change register (release-notes required)

1. **Lower tail latency under concurrency** when `AIRLOCK_PII_ENABLED`: PII no
   longer blocks the event loop. Single-request latency is unchanged; concurrent
   throughput improves. No output change.
2. **No wire/behavior change** from the ACL, decoupling, or race fix — pure
   internal refactors. Response headers, attribution, logs, and guard decisions are
   byte-identical.
3. **One file to re-verify on LiteLLM upgrades** (`litellm_adapter.py`) — call this
   out so operators know where the version-coupling now lives.

## Test surface

- **ACL parity:** for a representative set of responses (streaming + non-streaming,
  each provider-ambiguity case from §3.7), the served-by/region/mutations headers
  and attribution are identical before vs after the ACL extraction.
- **Presidio offload:** redaction output identical; a concurrency test proves
  requests no longer serialize behind Presidio.
- **Text-cache correctness:** keyword/guardian see the **post-PII-redaction** text
  (the order subtlety), and extraction runs once per request.
- **Import-cycle guard:** an automated check fails if `fast/`↔`guardrails/`
  re-couples.
- **Bootstrap order:** middleware install order asserted (batch → admin → header →
  LiteLLM).
- **threat_score race:** a concurrent-same-client test shows no lost update
  (deterministic via injected timing or a stress loop).
- **client_id parity:** the consolidated extractor matches all 3 legacy paths on a
  golden input set.

## Decisions taken (recorded)

- **Scope (2026-06-25, from audit):** 0.5.3 = the audit's Tier 1 latency + Tier 2
  structural items **not already covered by 0.5.1**. Budget unification (R1/R3/R6),
  DualCache/STORE-seam, and FIX-1 stay in **0.5.1**; bulkhead/isolation + the
  `StateProvider`/`state.py`-split groundwork go to **0.5.4**.
- **ACL is the anchor pack** — isolating LiteLLM internals is the highest-leverage
  coupling fix and a prerequisite for safe LiteLLM upgrades.
