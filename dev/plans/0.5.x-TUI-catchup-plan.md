# Plan — Bring the airlock TUI up to 0.5.x

## Context

The TUI was last meaningfully updated in 0.5.0 (served-via column + clear-quarantine
keybinding). Since then 0.5.1 / 0.5.2 / 0.5.4 shipped with **no** TUI work, so several
operator-relevant capabilities exist in the proxy but are invisible in the dashboard.
This plan closes the five real gaps, "done right" (correct data seam + usable display,
not minimal patches).

**Hard constraint — latency.** airlock is latency-sensitive. The 0.5.4 event bus
(`RequestEvent` + `RequestRecorder`) is an **in-process, synchronous** fan-out on the
request hot path with **no IPC** — the TUI must never subscribe to it. The TUI stays a
separate process that consumes airlock through three off-hot-path seams only:
JSONL tail, the admin HTTP read views, and (for capability) `/v1/models`.

### Import boundary (load-bearing rule)
- ✅ TUI MAY reference the canonical field contract from
  `airlock/callbacks/projections.py` / `airlock/callbacks/request_event.py`
  (`RequestEvent` is a side-effect-free frozen dataclass).
- ❌ TUI MUST NOT import `airlock/callbacks/recorder.py` — its module-level
  `_self_register()` (recorder.py:65) installs the live LiteLLM callback. Importing it
  from the TUI process is forbidden.
- This is why gap #3 parses JSONL against the **canonical projection field names**, not
  ad-hoc legacy `airlock_*` scraping.

### Production safety
All manual testing runs against a **separate dir + port** with copied config — never the
live service / `:4000` / `:8090` ([[airlock-production-safety]]).

## Pack A — Mutation ledger + served/attribution (gap #3)  · TUI-only, render-only

Cheapest, highest-transparency win; validates the parse-to-canonical-schema approach.

- **`airlock/tui/screens/guards.py`** — the live request stream already parses JSONL
  into `FlowEntry` and has a tabbed detail (Signals / Pipeline / Raw JSON / Tool Result).
  - Add a **Mutations** detail tab rendering the `mutations` ledger from the record:
    one row per entry showing `field`, `op`, `before`→`after`, `stage`, `source`,
    `reason`, `count`/`category`. Redactions render as `field redacted(N)` (count only,
    never content) — mirror the header allowlist policy in
    `docs/reference/response-headers.md`.
  - Surface `served.provider` / `backend_kind` and `attribution` ("served" vs
    "inferred") in the stream row and/or Pipeline tab.
- Parse against the field names emitted by `project_enterprise`
  (`airlock/callbacks/projections.py:24-51`), not legacy keys.
- No proxy change.

## Pack B — Rate-limit headroom + per-provider spend (gaps #1, #2)  · admin endpoint + Overview

Live-only data; surfaced via the existing admin read surface (off the hot path, behind
the admin perimeter). **Prereq: `admin.enabled: true`** (same prereq as the existing
clear-quarantine keybinding).

- **Proxy — `airlock/admin/http.py`** — extend `_view_providers()` (currently returns
  quarantine/cooldown/half_open/last_reason) to add per provider:
  - `ratelimit`: from `store.all_provider_ratelimits()` (`remaining_tokens`,
    `remaining_requests`, limits, reset) — `ProviderRateLimitState`,
    `airlock/fast/state.py:446`.
  - `spend`: from `store.get_provider_spend(name).recent_spend(...)`
    (`airlock/fast/state.py:420/963`) plus the configured cap from
    `router_settings.provider_budget_config` so the TUI can show "% of cap".
  - Stays `GET /airlock/admin/providers`, scope `admin:read`, no audit record, snapshot
    under the existing StateStore lock — never blocks request processing; perimeter
    "never raises" (CC-10) preserved.
- **TUI — `airlock/tui/admin_client.py`** — add a GET helper (mirror the existing
  `admin_post` urllib pattern at `admin_client.py:32`) e.g. `get_providers(...)`.
- **TUI — `airlock/tui/screens/overview.py`** — add **Headroom** (remaining req/tok) and
  **Spend** ($ + % of cap) columns to the Providers table, merged onto the existing
  JSONL-derived rows. Degrade gracefully (blank) when admin is disabled/unreachable.
- **TUI — `airlock/tui/alert_engine.py`** — revive the dormant `_check_provider_budget`
  no-op (`alert_engine.py:161`): fire an info/warning alert when
  `spend ≥ budget_warn_ratio × cap` (unified `airlock_settings.budget_warn_ratio`,
  default 0.8).

## Pack C — Capability discovery + deprecation (gap #4)  · TUI fetch of `/v1/models`

- **TUI** — add a capability fetch: `GET /v1/models` (auth with the configured master
  key), read the additive `airlock` object per model (`endpoints`, `underlying`,
  `region`, `deprecated`).
- **`airlock/tui/screens/test.py`** and **`airlock/tui/screens/advisor.py`** — the model
  selectors currently read `model_list` from `config.yaml`. Enrich them: flag
  `deprecated: true` aliases with a warning, and show `endpoints` (chat/batch) + `region`.
- Optional small capability panel (per selected model). Public endpoint — no admin dep.

## Pack D — Batch jobs view with live status (gap #5)  · proxy plumbing + new TUI view

Largest pack (live polling chosen). Two parts:

- **Proxy/gateway — `airlock/batch/`** — add a per-adapter `get_batch_status(job)` for
  each backend (Gemini / Mistral / OpenAI / Vertex), and expose a **read-only** status
  route (e.g. `GET /v1/batches/{id}` or an admin read route) that the gateway services by
  querying the provider's batch API. This is a separate read path triggered by the TUI
  poll — **not** invoked from the inference request path; must be throttled and
  bounded-concurrency so it never contends with the hot path.
- **TUI — new Batch pane** (or a dedicated section): build the job list/history by
  aggregating JSONL batch records by `batch_id` (`write_batch_record`,
  `enterprise_logger.py:280`; `tail_jsonl` already ingests `call_type="batch"`), then
  **poll the new status endpoint for non-terminal jobs** to show live state, row counts,
  and errors.

## Build order & rationale
1. **Pack A** (#3) — pure TUI, no proxy change, validates schema-parse approach.
2. **Pack B** (#1, #2) — one admin-endpoint extension serves both; adds Overview columns + revives the budget alert.
3. **Pack C** (#4) — TUI-side fetch + selector enrichment.
4. **Pack D** (#5) — biggest; proxy status plumbing + new view, sequenced last.

Each pack is independently shippable and testable.

## Latency / safety guardrails (apply to every pack)
- No TUI import of `airlock.callbacks.recorder` (hot-path callback). Schema names only.
- All new proxy reads (Pack B views, Pack D status route) are off the request hot path,
  snapshot under existing locks, and follow the perimeter "never raises" posture.
- Pack D provider polling is on-demand, throttled, non-terminal-jobs-only, bounded
  concurrency — must not block or contend with inference.

## Verification

  status adapters + route; revived budget-alert rule.

  env failure is unrelated).
- **Manual, isolated** ([[airlock-production-safety]]): launch a throwaway proxy on a

  each new surface populates and degrades gracefully when admin is disabled.
