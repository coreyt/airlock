## P-2b — ⚠️ SPLIT — Strict model resolution

**Shipped in 0.5.6:** the dropped-qualifier guard. `resolve()` refuses any fuzzy match
that would ignore a non-numeric token the caller supplied, and `suggest()` returns
ranked candidates with cost tier. Zero blast radius — the refused names
(`gpt-5.6-mini` etc.) only became reachable when 5.6 was added.

**Moved to 0.5.7:** wiring the *helpful response*. Today a refusal leaves the unknown
name in `data["model"]` and litellm returns its own generic error, so the caller gets
**no suggestion at all** despite `suggest()` already computing one. Unblocked and
additive — it changes the error body, not what is refused.

**Remains here (blocked):** extending refusal to *pre-existing* aliases — the cross-tier
rule. That one does change behaviour for names that resolve today, so it needs the
P-6c window like P-2 does.

**Status:** new; design §4.2. Replaces the withdrawn guess-guard approach.

**Files:** `airlock/fast/model_alias.py`, `airlock/proxy_errors.py`,
`airlock/transparency.py`, `airlock/callbacks/model_override_headers.py`.

Sibling of P-2: same principle (**don't substitute silently**), applied to model names
instead of `reasoning_effort`. Ships behind the same warn-only window (P-6c).

### Changes

- Fuzzy matching becomes a **diagnostic**, not a router:
  - exact match → serve;
  - near-match **within the same cost tier** → serve, log the resolution (keeps typo
    tolerance and dated snapshots like `gpt-5.6-sol-2026-07-09` working);
  - near-match **crossing a cost tier** → **reject**, using the score to *suggest*;
  - nothing above `_WARN_THRESHOLD` → reject, list available models.
- **404 + `code: model_not_found`**, OpenAI-shaped so existing SDK error paths work.
- `error.message` must be **self-sufficient prose** — most clients surface only that.
  The structured `error.airlock.suggestions` block (model, score, tier, price) is a
  bonus, not the payload.
- Add `X-Airlock-Model-Suggestion` using the established `;`-joined `key=value`
  grammar — reuse the `_mutation_token` serializer, do not write a second one.
- Suggestions are **derived** from the existing scorer + tier tables, never hardcoded.
- **Never suggest a model the caller is not entitled to** — respect per-client catalog
  filtering. A helpful error must not become a capability-disclosure channel.
- Threshold constants keep their values, change meaning: 0.50 now gates *same-tier
  auto-routing*; 0.35–0.50 becomes *suggestion* territory.

### Tests

- [ ] `gpt-5.6-sol-2026-07-09` → still auto-routes (same tier); no rejection.
- [ ] A same-tier typo still auto-routes — the convenience feature is not lost.
- [ ] `error.message` alone contains the suggestion and the available models.
- [ ] `X-Airlock-Model-Suggestion` grammar matches `_mutation_token` byte-for-byte.
- [ ] Suggestions never include a model outside the caller's permitted catalog.
- [ ] Warn-only mode: identical routing to today, but WARNING log + advisory header.

### Exit criteria

A client that guesses a non-existent model name is told exactly what to send instead,
and is never silently billed for a model it did not ask for.

## P-6 — `X-Airlock-Model-Alias` response header

> ⚠️ **Now covers a live gap.** 0.5.6 advanced `gemini-flash-lite` -> 3.1-flash-lite and
> `gemini-pro`/`gemini-flash` to the 3.x line. The `served=` disclosure that was
> supposed to make such advances visible does not exist yet, so callers are on
> different models — at different prices — and have not been told. This was accepted
> knowingly to hit the 2026-07-23 deadline; it should not stay unshipped.


**Status:** new capability; see design §6.

**Files:** `airlock/transparency.py`, `airlock/callbacks/model_override_headers.py`,
**plus** `config.yaml` + `airlock/cli/templates/config.yaml` (the successor map),
`airlock/fast/settings.py` or `router.py` (map loading/validation, following the
`cost_tiers` pattern), `tests/test_config_consistency.py` (map validation), and docs.

**Depends on:** P-4 (successor targets must exist in `model_list`) **and** the
successor-map config plumbing below — the original "depends on P-4 only" was wrong.

### Changes

Emit on OpenAI-family responses:

```
X-Airlock-Model-Alias: requested=gpt-5;served=openai/gpt-5.5;newer=openai/gpt-5.6-sol
```

**Grammar:** `;`-joined `key=value`, **no space after the separator**, each value via
`_header_safe`, byte-budgeted with the same `…+N more` truncation as
`mutations_header`. This matches `_mutation_token` exactly — **reuse the existing
serializer, do not write a second one.**

**Successor map** (design §6.1) — new optional `model_successors:` block in
`config.yaml`, `dict[str, str]` mapping **requested alias → advertised successor
alias**, loaded/validated on the `cost_tiers` pattern (env → config → empty default):

```yaml
model_successors:
  gpt-5: openai/gpt-5.6-sol
```

Keys and values must both exist in `model_list`, enforced in
`test_config_consistency.py` the same way fallback sources/targets are. Default empty:
with no map configured, the header is never emitted.

> ⚠️ **Look up the key using `metadata["airlock_request"]["requested_model"]`, not
> `data["model"]`.** The guardian already preserves the original request. By post-call
> time `data["model"]` may have been rewritten by routing, a cost-tier swap, or
> failover — keying off it would advertise a successor to a model the client never
> asked for. Load the map via an `AirlockSettings.model_successors` field following the
> `cost_tiers` loader at `airlock/fast/settings.py:209`.

Constraints:
- **Additive and advisory only — must never change routing.**
- Emitted through the existing `AirlockModelOverrideHeaders` post-call hook, gated by
  the transparency config, consistent with `served_headers` / `mutations_header`.
- `newer=` is **data-driven from a successor map**, not a hardcoded string, so it does
  not rot on the next release.
- Advertise the **provider-prefixed** form (`openai/gpt-5.6-sol`) — it matches
  `capability_record.underlying` and the 0.5.2 "stable served-by-explicit names"
  convention, and is directly callable given P-4's twins.
- Omit the header entirely when there is no successor. Never guess (same rule as
  `Served-By`, which omits rather than guessing an unknown provider).

### Tests

- [ ] Header present with correct k/v for `gpt-5`; absent when no successor is mapped.
- [ ] Routing is byte-identical with the header on and off — proves advisory-only.
- [ ] Respects the transparency config toggle.
- [ ] Header values are header-safe (reuse `_header_safe`).
- [ ] Fires for streaming and non-streaming alike (matching the `Mutations` rule).
- [ ] Grammar matches `_mutation_token` byte-for-byte (`;` separator, no space) —

- [ ] A `model_successors` key or value absent from `model_list` **fails**

- [ ] Empty/absent map → header never emitted, no exception.

## P-6c — ✅ SHIPPED IN 0.5.6 — Warn-only measurement window (GATES ENFORCEMENT)

**Status:** required. Design §13.

P-2 and P-2b both replace silent behaviour with rejections. The affected client
population **cannot be enumerated from the code** — only measured. Both therefore ship
**warn-only first** and enforce in the following release.

### Changes

Warn-only mode = today's behaviour exactly (same substitution, same translation, same
served model), plus:
1. **WARNING**-level structured log (never DEBUG), with a stable `event` discriminator.
2. The advisory response header the enforcing version would send.
3. A mutation-ledger entry, so it flows into `X-Airlock-Mutations` and the 0.5.4
   unified `RequestEvent` stream.

Required fields — this is the deliverable, not a side effect:

```
event=fuzzy_match_would_reject  requested= served= suggested= score=
                                from_tier= to_tier= client_id=
event=effort_would_reject       requested= translated_to= model=
                                supported= client_id=
```

- Both events **must** reach the existing observability path — not log-file-only, so
  they are countable with the same tooling as everything else.
- `client_id` present wherever client identity is known, so impact attributes to
  specific callers rather than a bare total.

### Tests

      served model across the whole matrix.
- [ ] Both events emit at WARNING with all required fields populated.
- [ ] Both events are retrievable from the event store, not just the log file.
- [ ] `client_id` is populated when client identity is known.
