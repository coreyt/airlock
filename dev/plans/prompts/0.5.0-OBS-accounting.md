## Design Decisions (resolved — Decision 4 / CC-T4 — do not redesign)
1. **Flag gate.** Read `attribute_accounting_to_served` once via
   `get_transparency_config().attribute_accounting_to_served` (default True). When **False**
   → preserve today's exact behavior (inferred everywhere). All new behavior is behind this flag.
2. **SUCCESS path (spend) — key off SERVED.** In `log_success_event`, when the flag is on:
   compute `served = attribute_served_backend(response_obj, cost_fallback=kwargs.get("response_cost"))`.
   - If `served` and `served.provider` → use `served.provider` as the spend key, and
     `served.response_cost` if it is not None, else `kwargs.get("response_cost", 0.0)`.
   - If `served` is None / `served.provider` is None → **fall back** to the current inferred
     provider + `kwargs` cost (back-compat; never blocks on a served read). `response_obj` is
     the **final** served response — attribute to it, not to fallback attempts.
3. **FAILURE / 429 path — key off the ERROR's provider, else inferred (CC-T4).** In
   `log_failure_event`, the quarantine/rate-limit provider should prefer the provider parsed
   from the exception (e.g. `getattr(exception, "llm_provider", None)`) when present, else the
   inferred provider. **Never block on a served read that does not exist** on failure (often
   no response/`_hidden_params`). Keep the existing `if provider and is_rate_limited:` gate.
4. **Do NOT change attempt/breaker counting.** Circuit-breaker counting of failed *attempts*
   keeps its existing per-attempted-provider behavior. Only spend (success) and quarantine
   provider-keying (failure) change. Don't touch the breaker state machine.
5. **CC-T4:** routing is unchanged — you only change which counter a number lands in.
