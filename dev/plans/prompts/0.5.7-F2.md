## Design Decisions (already resolved — you cannot infer these)

1. **The current `check()` never acquires.** It peeks `sem._value <= 0`
   (admission.py:125) — a burst detector, not a hard cap. Replace the peek with a real
   **non-blocking try-acquire**: `sem.locked()`-style guard then `sem.acquire_nowait()`
   is not a stdlib method — use the pattern `if sem._value <= 0: shed; else sem-acquire`
   done atomically, or acquire-then-check. **Acquire must never `await sem.acquire()`**
   (that queues; we shed instead). The RPM token-bucket half of `check()` is unchanged.

2. **Release must fire exactly once per admitted request, on every termination path**
   (normal, exception, timeout). There is **no release seam today.** The available
   once-per-request post-call hooks are in `model_override_headers.py`:
   `async_post_call_success_hook` (line 91) and
   `async_post_call_response_headers_hook` (line 28). A failure path also needs release
   — prefer LiteLLM's `async_log_success_event` / `async_log_failure_event` if the
   `CustomLogger` subclass exposes them, so both success and failure release. Whichever
   seam you choose, it must be reachable for BOTH outcomes; if only the success seam is
   reachable, add failure handling and record the gap in the design memo.

3. **Carry the acquired slot's identity** from pre-call to release. Stamp the client_id
   (and a boolean "we acquired") into `data["metadata"]` at admit time so the release
   hook knows which semaphore to release and that it must release exactly once. Guard
   against double-release (idempotent release keyed on a per-request flag).

4. **Fail-open is sacred.** Any error in acquire OR release logs a warning and does not
   break the request. A failed acquire fails OPEN (admit), never closed. A failed
   release must not leak a slot silently — log it.

5. **The gate stays synchronous where it can.** The RPM check is sync and fast. Only the
   concurrency acquire needs the async/await-free non-blocking primitive. Do not convert
   the whole guardian hook's structure — it is already `async` (guardian.py:234).
