## Design Decisions (already resolved — you cannot infer these)

### Capability schema — `capability_record(entry: dict) -> dict`
For a `model_list` entry, return EXACTLY these keys:
```python
{
  "airlock_provider": airlock_provider_for(entry),     # served-by token (reuse existing fn)
  "endpoints": endpoints_for(entry),                    # ["chat"] or ["chat","batch"]
  "underlying": (entry.get("litellm_params") or {}).get("model"),   # e.g. "gemini/gemini-3.5-flash"
  "region": (entry.get("litellm_params") or {}).get("vertex_location"),  # str for vertex_ai, else None
  "deprecated": <bool>,                                 # see rule below
}
```
- **`endpoints_for(entry)`** (design §4.5 — the single-source rule): start `["chat"]`;
  append `"batch"` **iff** the entry has an `airlock_batch` marker (truthy
  `entry.get("airlock_batch")`) **OR** it is a `vertex_ai/` model
  (`litellm_params.model` startswith `"vertex_ai/"`) **with a REGIONAL**
  `vertex_location` — i.e. `vertex_location` is set and `.lower() != "global"`. A bare
  `vertex_ai/` prefix is NOT sufficient; the current vertex entries use
  `vertex_location: global`, so they get **`["chat"]`** (NOT batch). This is the
  codex-BLOCK-3 anti-overclaim rule — do not weaken it.
- **`deprecated`** = `entry["model_name"].endswith(("-aistudio", "-vertex", "-batch"))`.
  This deterministically marks the 7 legacy suffix twins
  (`gemini-3.5-flash-aistudio`, `gemini-3.1-pro-aistudio`, `gemini-3.5-flash-vertex`,
  `gemini-3.1-pro-vertex`, `mistral-large-batch`, `mistral-small-batch`,
  `qwen36-27b-vllm-batch`) deprecated; every bare and `provider/` alias = not deprecated.
- `region` is `None` for non-vertex entries (no `vertex_location` key).
- Entries with no/empty `litellm_params.model` (none today, but be safe): return a
  record with `airlock_provider=None`, `endpoints=["chat"]`, `underlying=None`, etc.
  — do NOT crash. `smart`/cost-tier directives are not in `model_list`, so they're N/A.

### Injection seam — `proxy._prepare_runtime_config`
Inject computed `model_info` into the in-memory config BEFORE the temp config is
written (so the litellm child serves it natively on `/model/info`). After the config
dict is loaded (`proxy.py:160-161`) and alongside the other mutations:
```python
for entry in config.get("model_list", []) or []:
    if not isinstance(entry, dict):
        continue
    entry.setdefault("model_info", {}).update(capability_record(entry))
    changed = True
```
- Use `setdefault(...).update(...)` so a pre-existing `model_info` is preserved and our
  computed keys are written. (No entry has a model_info today.)
- This runs UNCONDITIONALLY (not env-gated) — capability is always published. It is fine
  that `changed=True` then always writes a temp runtime config; that is the existing
  behavior whenever any override fires.
- Import `capability_record` from `airlock.capability`. Watch for import cost/cycles —
  `capability.py` is pure and must not import `proxy`.
