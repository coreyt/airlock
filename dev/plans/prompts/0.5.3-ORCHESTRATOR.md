## Guardrails (do not violate)

- You do **not** edit source/test in the main checkout — delegate to a worktree
  implementer. Docs/STATUS edits are yours.
- **Production safety ([[airlock-production-safety]]):** never run against the live
  proxy/port.
- A pack whose diff changes any on-the-wire output (headers, attribution, logs,
  guard decisions) **other than the documented latency win** is a review **BLOCK** —
  these are behavior-preserving refactors.
- Never merge before `output.json` + codex PASS. Never override a BLOCK. Never
  `git push`/tag/amend without human approval.
- **Testing harness (parity oracle):** 0.5.3 adds no new client-facing surface, so
  `dev/smoketest/` is run **before vs after** the `ACL` extraction and the
  `served_header_client.py` output must be **byte-identical** (this is how AC-ACL is
  witnessed end-to-end); `LATENCY` reuses the same isolated instance for the
  throughput confirmation. If the optional event-bus pack changes any client-visible
  field, it **extends** the harness. A green isolated-instance run is required at
  sign-off.
- Protect context: hold conclusions; delegate large reads to `Explore`.
