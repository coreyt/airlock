## Design Decisions (resolved)

### 1. `project_s3(event) -> dict` (in projections.py)
Reproduce `AirlockS3Logger._build_record` (s3_logger.py:64-101) field-for-field from the
event, then apply `_redact_record`:
```python
from airlock.callbacks.enterprise_logger import _redact_record, _serialize
def project_s3(event):
    record = {
        "timestamp": event.timestamp,
        "success": event.success,
        "model": event.model,
        "user": event.user,
        "team": event.team,
        "request_id": event.request_id,
        "messages": event.messages,
        "response": _serialize(event.response_obj) if event.response_obj else None,
        "error": event.bare_exception_error,   # s3 emits str(exception)|None == event.bare_exception_error (§3.9)
        "duration_ms": event.duration_ms,
        **event.usage,
    }
    return _redact_record(record)
```
(Keep the exact key order of the old builder; s3's narrow set has NO guardrail_meta/
mcp/served/provider/record_type — do not add them.)

### 2. `AirlockS3Logger.record_event(self, event)` (s3_logger.py)
`self._append(project_s3(event))` — reuse the existing `_append`/buffer/`_flush` path
unchanged. Then DELETE `_build_record` and the 4 callback methods. KEEP `_append`,
`_flush`, `flush()`, the buffer + lock, `proxy_s3_logger`, the `atexit` flush,
`AIRLOCK_S3_BUCKET` gate.

### 3. recorder.py — gated registration
After the enterprise (and fathom-if-enabled) registrations, add:
```python
if _env_flag("AIRLOCK_ENABLE_S3_LOGGER", default=False):
    from airlock.callbacks.s3_logger import proxy_s3_logger
    request_recorder.register(proxy_s3_logger.record_event, name="s3")   # normal sink (success+failure; NOT async_only)
```
s3 fires wherever the recorder fires (the recorder is on all 4 lists). The
`AIRLOCK_S3_BUCKET` write-gate is unchanged (s3 still discards when no bucket).
