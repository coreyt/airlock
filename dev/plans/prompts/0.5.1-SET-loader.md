## Design Decisions (already resolved — you cannot infer these)

1. **Read in place — no key moves.** `provider_budget_config` STAYS under
   `router_settings` (LiteLLM owns the key). `AirlockSettings` reads it FROM
   `router_settings.provider_budget_config`. cost_tiers stays top-level. Unification
   is *logical* (one loader), never a physical config move. Relocating a key is wrong.
2. **Budget parsing reuses LiteLLM primitives.** Parse the budget block with
   `from litellm.types.utils import BudgetConfig` and
   `from litellm.litellm_core_utils.duration_parser import duration_in_seconds`
   (verified importable in this repo's litellm 1.89.0). `BudgetConfig` maps
   `budget_limit → max_budget` and `time_period → budget_duration`; use
   `duration_in_seconds` to turn `"1d"` into a window in seconds. Tolerate both the
   `budget_limit`/`time_period` spelling AND the LiteLLM-native `max_budget`/
   `time_period` spelling (BudgetConfig handles this) — malformed entries fall back to
   skipping that provider (treat as no budget), logging a warning. Do not crash on a
   bad block.
3. **`fallbacks` shape.** `router_settings.fallbacks` is a list of single-key dicts:
   `[{claude-opus: [claude-sonnet, ...]}, {claude-sonnet: [...]}, ...]`. Convert to
   `dict[str, list[str]]`. (SET-unify wires this into the breaker; you only parse +
   expose it.)
4. **`0 ⇒ no enforcement` contract.** A budget of `0`/falsy means *no enforcement /
   unlimited / no swap* everywhere. Your loader must preserve `0` as a real configured
   value (do not drop or coerce `0` to a default). Document this on the field.
5. **Precedence is uniform `env > config > default` for every field**, with malformed
   env/config falling back to the next level (warn + continue), exactly like
   `_load_cost_tiers`. Env var names: `AIRLOCK_PROVIDER_BUDGETS` (JSON),
   `AIRLOCK_FAILOVER_MAP` (JSON), `AIRLOCK_COST_TIERS` (JSON), `AIRLOCK_SESSION_TTL`
   (int), `AIRLOCK_SMART_THRESHOLDS` (JSON 2-list), `AIRLOCK_BUDGET_WARN_RATIO`
   (float). Reproduce the EXACT current defaults from the READ-ONLY anchors.
6. **New `airlock_settings:` keys** (read-support only; SET-unify populates config):
   `airlock_settings.session_ttl`, `airlock_settings.smart_thresholds`,
   `airlock_settings.budget_warn_ratio`. Absent ⇒ default. (Today these are env-only.)

## `AirlockSettings` shape (suggested — adapt names to fit the codebase)

A frozen dataclass with at least: `provider_budgets: dict[str, float]` (USD limit per
provider; `0` preserved), `budget_windows: dict[str, float]` (seconds, parsed from
`time_period`, default 86400 / `"1d"`), `failover_map: dict[str, list[str]]`,
`cost_tiers: dict[str, list[str]]`, `session_ttl: int`, `smart_thresholds:
tuple[float, float]`, `budget_warn_ratio: float`. Plus `load_airlock_settings(config:
dict | None) -> AirlockSettings`, a module-level singleton + `configure_settings(config)`
setter and `get_settings()` getter (mirror how `transparency.py` /
`router.set_router_config` cache config at startup).
