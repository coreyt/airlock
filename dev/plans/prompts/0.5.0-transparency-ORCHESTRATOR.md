worktrees (TDD), and every pack is reviewed by **codex** before it merges. This

chat.

Drive the transparency workstream from its current state (design complete &

`feat/0.5.0-resilience-admin`, reviewed, and CLOSED**, halting to the human (HITL)
at the gates named below. Maintain the live STATUS board across the run.

## Per-pack loop (Phase E)

Pack order (critical path `OBS-core → OBS-served → OBS-headers`; `OBS-ledger` ∥
`OBS-served`; then `OBS-log`, `OBS-accounting`, `OBS-metrics-tui`):

```
OBS-core → OBS-served ─┬→ OBS-headers
           OBS-ledger ─┤→ OBS-log
                       └→ OBS-metrics-tui
OBS-served ─────────────→ OBS-accounting
```

For each pack, run this state machine (advance only when the prior **witness**
exists — runbook §1.5):

1. **Pre-flight.** Confirm dependencies are CLOSED on the branch. Re-verify the
   pack's file:line anchors against the current branch HEAD (they drift) before
   authoring.
2. **Author the prompt.** Fill `dev/plans/prompts/SLICE-TEMPLATE.md` →
   `dev/plans/prompts/0.5.0-<PACK>.md`: scope, `{{WORKTREE_PATH}}`, `{{BRANCH}}`,
   `{{BASE_COMMIT}}`, `{{TARGET_TESTS}}`, `{{MODIFY}}`, `{{READ_ONLY}}` (with line
   ranges), `{{DO_NOT_TOUCH}}`, the **already-resolved design decisions** (from the
   design note — the implementer cannot infer them), the CC-Ts this pack
   owns/consumes, a **no-network** TDD test plan, and the DoD. Follow the
   SLICE-TEMPLATE authoring checklist (name companion artifacts, not just the
   primary file).
3. **Spawn the implementer** (`implementer` subagent, `isolation: worktree`). It
   works **only** in its worktree, follows RED → READ → GREEN → LINT → COMMIT →
   CLOSE, writes `dev/plans/runs/0.5.0-<PACK>-output.json` last, and returns the
   REPORT. It does **not** spawn subagents and never touches the main checkout. TDD
   is mandatory — RED tests committed on their own first.
4. **Gate on the witness.** Require `0.5.0-<PACK>-output.json` present and commits
   past baseline. If absent or the REPORT reports a blocker → the pack is FAILED;
   triage (fixer pack or abandon), never merge.
5. **Codex review** (`dev/agent-harness-reference.md` §3.1):
   `codex exec --model gpt-5.5 -c model_reasoning_effort=high --sandbox read-only
   --cd <worktree>` over the pack's diff/files (point at FILES; stdin via
   `printf '%s\n' "$PROMPT" | codex exec … -`; never let codex write). Promote the
   verdict to `dev/plans/runs/0.5.0-<PACK>-review-<ts>.md` with an
   `## Orchestrator triage`. **Map to the merge gate: PASS → merge; CONCERN → fix
   or orchestrator-override with rationale (never override BLOCK); BLOCK → fixer
   pack + re-review.** Use the `code-reviewer` subagent (model `sonnet`) **only**
   if codex is unavailable, and note the fallback on the board.
6. **Merge → close → clean.** After PASS (or cleared fixes): merge worktree →
   `feat/0.5.0-resilience-admin` (verify green target tests + no-network suite, run
   in background or `| tail -5`, never the full suite in foreground), write the
   pack's CLOSED block on the STATUS board with the promoted verdict + commit shas,
   then remove the worktree.
7. **HITL halt where the ladder says so** (below). Do not proceed past a HITL gate
   without the human's confirmation.

Update `dev/plans/runs/STATUS-0.5.0-transparency.md` on **every** transition (one
docs commit per transition). Keep the acceptance scoreboard (UN-19/UN-20) current.

## HITL gates (halt and ask the human)

- **After OBS-served merges** — new `X-Airlock-Served-By`/`-Region` headers on
  every response (a contract clients may key on). Ask the operator to smoke-test a
  native-alias call and a gateway-alias call (e.g. `vertex_ai` vs `gemini`/AI-Studio,
  and a Bedrock-routed model) and confirm the headers report the real backend.
- **After OBS-accounting merges** — it changes which provider spend/quarantine is
  debited to. Ask the operator to validate accounting + dashboards on real traffic
  before GA. (Default `attribute_accounting_to_served: on`, documented as a bugfix.)
- **Release sign-off** — folded into the 0.5.0 release HITL, after all seven
  transparency packs are CLOSED and the smoke-tests pass.

## Guardrails (do not violate)

- You do **not** edit source/test files in the main checkout — delegate to an
  implementer in a worktree. Docs/STATUS edits on the branch are yours.
- Never merge before `output.json` exists and codex has PASSed (or a fix cleared a
  BLOCK and re-review PASSed).
- Never override a codex **BLOCK**; a BLOCK a fixer can't clear, or fixes past a
  small bound, **halts to HITL**.
- Never `git push`, force-push, or amend without explicit human approval.
- Protect your context: hold conclusions, not raw agent/codex output; delegate
  large reads to `Explore`.
- Respect CC-T1…CC-T7 as cross-pack invariants — flag any pack whose diff would
  break one (e.g. a redaction record carrying a value violates CC-T2; a request-path
  behavior change outside OBS-accounting violates CC-T4; a streaming served-by read
  from the wrong hook violates CC-T6) as a review BLOCK.
