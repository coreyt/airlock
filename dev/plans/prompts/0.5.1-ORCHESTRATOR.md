## Guardrails (do not violate)

- You do **not** edit source/test files in the main checkout — delegate to an
  implementer in a worktree. Docs/STATUS edits on the branch are yours.
- **Production safety ([[airlock-production-safety]]):** never run against the live
  proxy/port; all smoke-tests use a separate dir+port + copied config.
- `provider_budget_config` **stays in `router_settings`** (LiteLLM owns the key) —
  unification is **logical** (read-in-place), never a physical key move. A pack that
  relocates it is a review BLOCK.
- Never merge before `output.json` exists and codex PASSed. Never override a BLOCK.
  Never `git push`/tag/amend without explicit human approval.
- **Testing harness:** any pack that changes a client-facing surface (here:
  `X-Airlock-Budget-State`, the `X-Airlock-Model-Override` swap point, failover
  target) **must extend `dev/smoketest/`** with an assertion for it; the
  `STORE-seam` restart-durability check runs through the same isolated instance. A
  green isolated-instance run (separate dir+port, copied config) is required at
  sign-off. A client-facing change with no harness coverage is a review **BLOCK**.
- Protect your context: hold conclusions, not raw agent/codex output; delegate
  large reads to `Explore`.
