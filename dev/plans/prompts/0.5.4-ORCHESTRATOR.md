Pack order (the IMPL-* set's shape is set by DECIDE):


Standard state machine per pack (advance only on the prior witness):

1. **Pre-flight** — dependencies CLOSED; anchors re-verified vs HEAD.
2. **Author the prompt** — `SLICE-TEMPLATE.md` → `dev/plans/prompts/0.5.4-<PACK>.md`
   with the resolved decision, scope, worktree/branch/base, target tests,
   MODIFY/READ_ONLY, DO_NOT_TOUCH, a **no-network** TDD plan, and the DoD. IMPL
   prompts **must** encode the implementation guardrails (fail-open, shed with

3. **Spawn the implementer** (`implementer`, `isolation: worktree`): RED → READ →
   GREEN → LINT → COMMIT → CLOSE, writes `dev/plans/runs/0.5.4-<PACK>-output.json`

4. **Gate on the witness** — else FAILED.
5. **Codex review** — over the diff → `dev/plans/runs/0.5.4-<PACK>-review-<ts>.md`.
   PASS → merge; never override BLOCK.
6. **Merge → close → clean** — green target tests + no-network suite, CLOSED block

### Pack scopes (from the plan)

- **ENABLE-stateprovider** — replace the module-global `store = StateStore()` with
  an injected `StateProvider` bound at startup (lazy-fallback for back-compat).
  Golden-equivalent to the global store. (Audit Tier 3 #7; prerequisite for any
  shared-store/multi-worker path.)
- **ENABLE-statesplit** — split the `state.py` god-object into
  core/spend/persistence/mcp modules. Behavior-preserving; full suite green; no
  public-API change. (Audit Tier 3 #9.)
- **IMPL-*** — the chosen mechanism. For C1: per-client concurrency gate +
  token-bucket in the guardian pre-call **consuming `PrioritySignal`**, with
  metrics (`airlock_admission_*`), an `X-Airlock-Admission` header, and a
  `429 + Retry-After` shed policy. For C2/C3: `--num_workers` + shared-store wiring
  (depends on 0.5.1 FIX-1 checkpoint-in-child).
- **DOCS** — UN-23/UN-24 in `dev/user-needs.md`; finalize
  `design-bulkhead-isolation.md` as as-built; ops guide stating **what is and is
  not isolated**; changelog + behavior-change register.

## Guardrails (do not violate)

- You do **not** edit source/test in the main checkout — delegate to a worktree
  implementer. Docs/STATUS/`dev/notes` edits are yours (the decision memo may be
  delegated to a non-implementer agent that writes `dev/notes/` only).
- **Production safety ([[airlock-production-safety]]):** all experiments + load
  tests on a separate dir+port; the live `:4000` service is never touched.
- **Isolation must not become a new outage:** any pack whose gate can block a
  request on internal error (instead of failing open), can hang instead of shedding,
  adds >1ms baseline, or introduces a second parallel state store is a review **BLOCK**.
- Throwaway spikes are **never merged**. Never author a Phase-E pack before DECIDE.
- Never merge before `output.json` + codex PASS. Never override a BLOCK. Never
  `git push`/tag/amend without human approval.
- **Testing harness:** the chosen mechanism's new client-facing surface (likely
  `X-Airlock-Admission` + the `429 + Retry-After` shed) **must** be covered by an
  extension to `dev/smoketest/` — the noisy-neighbor scenario *is* the UN-23 load
  test. The IMPL-* pack authors it; `DOCS` finalizes it; the client keeps its
  production-safety guards. If `DECIDE` picks a mechanism with no new surface, the
  harness is re-specified at `DECIDE` (parity oracle as in 0.5.3). A green
  isolated-instance run covering the chosen surface is required at sign-off; a new
  client-facing surface with no harness coverage is a review **BLOCK**.
- Protect context: hold conclusions; delegate large reads/benchmarks to subagents.
