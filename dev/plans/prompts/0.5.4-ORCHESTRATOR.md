## Phase E — per-pack loop

Pack order (`EVENT` is the anchor; the `MIGRATE-*` packs depend on it and all touch

sequential or small disjoint batches**, one sink per worktree):

```
EVENT → { MIGRATE-enterprise, MIGRATE-fathom, MIGRATE-s3, MIGRATE-sql,
          MIGRATE-sidechannels } → VERIFY → DOCS
```

For each pack run the standard state machine (advance only on the prior witness):

1. **Pre-flight** — dependencies CLOSED on the branch; re-verify the pack's anchors
   (builder symbols / call sites) vs HEAD.
2. **Author the prompt** — `SLICE-TEMPLATE.md` → `dev/plans/prompts/0.5.4-<PACK>.md`:

   MIGRATE prompt **must** require a **golden field-for-field equivalence test**
   (sink output identical before vs after) and **delete that sink's
   `_build_record()`**.
3. **Spawn the implementer** (`implementer`, `isolation: worktree`): RED → READ →
   GREEN → LINT → COMMIT → CLOSE, writes `dev/plans/runs/0.5.4-<PACK>-output.json`

4. **Gate on the witness** — `output.json` + commits past baseline, else FAILED.
5. **Codex review** — over the diff → `dev/plans/runs/0.5.4-<PACK>-review-<ts>.md`

transition). Keep the acceptance scoreboard current.

## Guardrails (do not violate)

- You do **not** edit source/test in the main checkout — delegate to a worktree
  implementer. Docs/STATUS/`dev/notes` edits on the branch are yours.
- **Production safety ([[airlock-production-safety]]):** never run against the live
  proxy/port; all smoke runs use a separate dir+port + copied config.
- **Behavior-preserving is the contract.** A pack whose diff changes any sink's
  emitted record (fields or values) without an intentional, registered,
  smoke-covered change is a review **BLOCK**. Deleting the four `_build_record()`s is
  the deliverable, not optional. No second source of truth for any field; no
  measurable hot-path latency added.
- **Testing harness:** 0.5.4 expects **no** logged/served field change, so
  `dev/smoketest/` is the **parity oracle** (before/after identical). If a field
  shape unavoidably changes, the responsible MIGRATE pack **must** extend the
  harness + register the change; a client-visible change with no harness coverage is
  a review **BLOCK**. A green isolated-instance run is required at sign-off.
- Never merge before `output.json` + codex PASS. Never override a BLOCK. Never
  `git push`/tag/amend without explicit human approval.
- Protect your context: hold conclusions, not raw agent/codex output; delegate large
  reads to `Explore`.
