### B. `AdmissionStore` — token-bucket counter (SpendStore pattern)

New class in `airlock/fast/admission.py`. Stores **integer request counts** (not µ$)
in a time-bucketed DualCache — same pattern as `SpendStore` but semantics differ:
- `record_request(client_id, timestamp)` → increments a 1-second bucket
- `recent_count(client_id, window_s, now)` → sums in-window buckets (int)
- `reset(client_id)` → clears all buckets for a client (for testing)
- Backed by `DualCache(InMemoryCache(...))` lazily initialized, same TTL-FIX-3 pattern
- Lock is the store's own `threading.RLock` (not shared with StateStore)
- **No spend semantics** — no µ$ conversion; counts are raw integers

### C. `AdmissionGate` — the hot-path gate (also in `admission.py`)

```python
class AdmissionGate:
    def __init__(self, config: AdmissionConfig, store: AdmissionStore):
        self._config = config
        self._store = store
        self._semaphores: dict[str, asyncio.Semaphore] = {}
        self._sem_lock = threading.Lock()

    def _get_semaphore(self, client_id: str) -> asyncio.Semaphore:
        with self._sem_lock:
            if client_id not in self._semaphores:
                self._semaphores[client_id] = asyncio.Semaphore(
                    self._config.concurrency
                )
            return self._semaphores[client_id]

    def check(self, client_id: str, boost: bool, now: float) -> tuple[bool, float]:
        """
        O(1) admission check. Returns (allowed, retry_after_s).
        retry_after_s is the time until the next request window opens (≥0).
        Fails open on any internal exception (logs warning, returns (True, 0.0)).
        """
        try:
            rpm_cap = int(
                self._config.rpm * (self._config.boost_multiplier if boost else 1.0)
            )
            window_s = 60.0
            count = self._store.recent_count(client_id, window_s, now)
            if count >= rpm_cap:
                # Retry-After: seconds until the oldest in-window request expires
                retry_after = max(0.1, window_s - (now % window_s))
                return False, retry_after
            self._store.record_request(client_id, now)
            # Concurrency check (non-blocking peek — no await here)
            sem = self._get_semaphore(client_id)
            if sem._value <= 0:
                return False, 1.0   # retry after ~1s for concurrency shed
            return True, 0.0
        except Exception:
            logger.warning("admission gate error — failing open", exc_info=True)
            return True, 0.0
```

**Fail-open invariant:** any exception inside `check()` logs a warning and returns
`(True, 0.0)`. **Never block a request because the limiter errored.**

## Scope discipline

- **Fail-open is non-negotiable.** Any exception in the gate path must log and
  return `(True, 0.0)`. If your implementation can raise from `check()` without
  the fail-open wrapper catching it, that is a BLOCK.
- **No hang.** The `check()` method is synchronous (no `await`). Concurrency
  check is a non-blocking peek at `sem._value` — not an `async with sem` acquire.
  The gate does NOT hold a semaphore slot; it checks available slots as a guard
  and rejects if none are available. (A more sophisticated implementation using
  `asyncio.Semaphore` acquire would require an `async` gate and context-manager
  lifetime — out of scope for this pack. The peek approach is conservative and
  correct for the shed semantic.)
- **Off-by-default.** If `admission.enabled` is `False` (the default), the
  `_admission_gate` module variable is `None` and the gate block in guardian.py
  is a single `if _admission_gate is not None:` check — essentially free.
- **<1ms budget.** `check()` must complete in well under 1ms. The Phase 0
  benchmarks measured the gate prototype at p99=1.5µs. Do not add I/O or heavy
  computation inside `check()`.
- **No new parallel state store.** `AdmissionStore` uses the same DualCache
  pattern as `SpendStore`. It does NOT introduce a second in-memory cache with
  its own TTL management separate from the existing seam.
- **No double-count against breaker.** `AdmissionStore` is keyed by `client_id`
  only, never by model. It does not read `ModelState`.

## Implementation guardrails (enforced by the codex review)

A reviewer will BLOCK the merge if any of the following:
- Gate can raise from `check()` without the fail-open wrapper catching it.
- Rejected request returns without a `retry_after > 0` value.
- `retry_after` is hardcoded or zero for the RPM-shed path.
- `_admission_gate is None` path adds measurable overhead (it must be a cheap
  attribute check, not a function call or import).
- `AdmissionStore` shares `StateStore._lock` (it must have its own lock).
- The gate is on by default (`enabled=True` in defaults).
- `compute_priority` is called twice on the same request (once for the gate,
  once at Step 4) — call it once and pass the result to both.
