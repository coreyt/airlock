## Design Decisions (resolved)

### 1. `project_sql(event) -> dict` (projections.py)
Reproduce `AirlockSQLLogger._build_record` (sql_logger.py:90-128) field-for-field from the
event — the NARROW set with **JSON-STRING** messages/response (sql stores strings, NOT
objects), bare error, NO `_redact_record`. Use the SAME `_serialize` `sql_logger` uses
(reuse its import) so `json.dumps(..., default=_serialize)` output is byte-identical:
```python
record = {
    "timestamp": event.timestamp,
    "success": event.success,
    "model": event.model,
    "user": event.user,
    "team": event.team,
    "request_id": event.request_id,
    "messages": json.dumps(event.messages, default=_serialize),
    "response": json.dumps(_serialize(event.response_obj), default=_serialize) if event.response_obj else None,
    "error": event.bare_exception_error,
    "duration_ms": event.duration_ms,
    **event.usage,
}
```
(Match the OLD builder's exact key order. NO guardrail_meta/mcp/served/provider/record_type,
NO redaction. Verify the exact `json.dumps` call shape against the current `_build_record`
:117-118 — reproduce it verbatim.)

### 2. `AirlockSQLLogger.record_event(self, event)` (sql_logger.py)
Mirror the old `log_success_event` body but source from the event: build `project_sql(event)`
then `self._insert(record)` (keep `_insert`'s `_ensure_initialized` + engine/table guard +
exception swallow exactly). Then DELETE `_build_record` + the 4 callback methods. KEEP
`_insert`, `_ensure_initialized`, `_get_table`, engine/table, `proxy_sql_logger`, the
`AIRLOCK_SQL_URL`-disabled behavior.

### 3. recorder.py — gated registration
After the s3 gate, add:
```python
if _env_flag("AIRLOCK_ENABLE_SQL_LOGGER", default=False):
    from airlock.callbacks.sql_logger import proxy_sql_logger
    request_recorder.register(proxy_sql_logger.record_event, name="sql")   # normal sink (success+failure; NOT async_only)
```
