## Design Decisions (already resolved by the Phase-D design gate — you cannot infer these)

1. **The ACL is a PURE EXTRACTION. Zero behavior change.** Same values, same
   streaming-vs-non-streaming branches, same "unknown provider ⇒ omit the header,
   don't guess" rule. Response headers, attribution, logs, and middleware install
   order must be byte-identical before vs after. Any on-the-wire change is a BLOCK.
2. **ACL surface** (settle exact names yourself, but cover all of these):
   - `resolve_proxy_app()` → the LiteLLM proxy ASGI app (wraps
     `sys.modules.get("litellm.proxy.proxy_server")` + `.app`), or `None` if unresolved.
     Migrate health.py, proxy_errors.py, docs.py, models_seam.py, batch/middleware.py,
     admin/http.py to call this instead of poking `sys.modules` directly.
   - `install_asgi_middleware(app, factory)` (or equivalent) that encodes the
     **`middleware_stack is None` (use `add_middleware`) vs built-stack (wrap it)**
     branch **once**. models_seam / batch/middleware / admin/http currently each
     duplicate this branch — they must call the ACL helper, passing their own
     middleware factory. (Q2: ACL owns this MECHANISM; install ORDER stays in the
     callers / is the DECOUPLE pack's `proxy_bootstrap.py` concern — do NOT change
     order here.)
   - `_hidden_params(response) -> dict` accessor.
   - served-backend / provider accessor:
     `served_provider(response)` = `hp.get("custom_llm_provider") or getattr(response, "custom_llm_provider", None)`
     (the streaming-wrapper `vertex_ai_beta` hardcode case must behave exactly as today).
   - cost accessor: `response_cost(response, fallback=None)` reading `hp.get("response_cost")`
     and falling back to `fallback` (Q1: callers pass `cost_fallback=kwargs.get("response_cost")`).
   - additional-headers read accessor + a write helper for model_override_headers's
     `hidden_params.setdefault("additional_headers", {}).update(...)` path.
   `transparency.attribute_served_backend` and `fast/monitor` and
   `model_override_headers` then call these accessors instead of touching
   `_hidden_params`/`custom_llm_provider`/`response_cost`/`additional_headers` directly.
3. **Single home for version assumptions (Q4).** The "LiteLLM 1.89.0 `_hidden_params`
   contract" note (currently embedded in `transparency.py` comments near the served-
   backend reader, ~line 181–195, and §3.7) moves into the `litellm_adapter.py`
   module docstring. Downstream modules stop carrying version assumptions. State in
   the docstring: "This is the ONE file to re-verify on a LiteLLM upgrade."
4. **Do NOT widen scope to a provider-dispatch registry.** The ACL isolates *reads
   of internals* only — not provider routing/dispatch.
