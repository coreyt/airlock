### B. Collision-safe `model_alias` (design Â§4.1) â€” THE load-bearing correctness fix

Today `load_from_config` (`model_alias.py:235-264`) inserts, for every entry, a

in `model_alias`, not left to LiteLLM.

Implement BOTH:

1. **Two-pass `load_from_config`:**
   - **Pass 1:** insert every explicit `model_name` key (`_exact[alias.lower()] = alias`)
     first. These are **authoritative and immutable** â€” never overwritten later.
   - **Pass 2:** add bare-stripped / version-stripped variant keys **only when** (a)
     the key is not already an explicit alias key, **and** (b) it is **unambiguous**

   `(provider_token, bare_body) â†’ alias` index plus, per bare_body, the set of
   providers it appears under. Provider token derived per entry from its
   `litellm_params.model` via the **shared classifier** (Â§C) â€” NOT the alias prefix.
