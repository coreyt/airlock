## Design Decisions (resolved)

### 1. `AirlockMetricsCallback.record_event(self, event)` (metrics.py)
Reproduce the success+failure callback behavior from the event:
```python
def record_event(self, event):
    if not _PROM_AVAILABLE:
        return
    model = event.model
    user = event.user or "unknown"          # metrics defaults user to "unknown" (§3.6); loggers leave None
    _metrics["requests_total"].labels(model=model, user=user,
                                      success="true" if event.success else "false").inc()
    if event.success:
        if event.start_time and event.end_time:
            duration_s = (event.end_time - event.start_time).total_seconds()   # from raw datetimes, NOT duration_ms
            _metrics["request_duration"].labels(model=model).observe(duration_s)
        _record_mutations_from_event(event.mutations)
```
- `user` uses the **"unknown"** default (matches metrics.py:183-187), NOT the logger
  convention. `requests_total` fires on BOTH paths; `request_duration`+mutations success-only.

### 2. Mutation counting from the event (the key subtlety)
Today `_record_mutations(metadata)` reads RAW `Mutation` objects via `getattr(m,"field")`/
`getattr(m,"op")`. But `event.mutations` is the **asdict'd** form (list of dicts with
`field`/`op` keys, from `dataclasses.asdict`). So add `_record_mutations_from_event(mutations:
list[dict])` that increments `mutations_total{field,op}` per item using **dict access**
(`m.get("field")`, `m.get("op")`), skipping when either is None — behavior-identical to the
old `getattr` version (same field/op values, same skip rule). Keep the old
`_record_mutations(metadata)` ONLY if something else still calls it; otherwise replace it
with the event-driven helper and update the one caller. (Confirm via grep that metrics'
success callback was its only caller.)

### 3. recorder.py — register metrics UNCONDITIONALLY (always-on, normal sink)
```python
from airlock.callbacks.metrics import metrics_callback
request_recorder.register(metrics_callback.record_event, name="metrics")   # always; normal sink (success+failure)
```
Place it after the enterprise registration (order doesn't matter — metrics reads no
monitor-mutated field). Then DELETE metrics' `_self_register` def + its module-level call
(metrics is now dispatched via the recorder, not self-registered — avoids double-emit).
