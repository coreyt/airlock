## Phase E — per-pack loop

Pack order (critical path; config-editing packs are **sequential** — they share
`config.yaml model_list`, so no parallel worktrees on it):

```
NAME-aliases → CAP-modelinfo → CAP-v1models → COMPAT-tests → DOCS
```

For each pack, run this state machine (advance only when the prior **witness**
exists — runbook §1.5):

1. **Pre-flight.** Confirm dependencies CLOSED on the branch. Re-verify the pack's
   anchors + Appendix A rows against branch HEAD before authoring.
2. **Author the prompt.** Fill `dev/plans/prompts/SLICE-TEMPLATE.md` →
   `dev/plans/prompts/0.5.2-<PACK>.md`: scope, `{{WORKTREE_PATH}}`, `{{BRANCH}}`,
   `{{BASE_COMMIT}}`, `{{TARGET_TESTS}}`, `{{MODIFY}}`, `{{READ_ONLY}}` (line
   ranges), `{{DO_NOT_TOUCH}}`, the **already-resolved** design decisions (from the
   design note — the implementer cannot infer them), the N-items this pack
   owns/consumes, a **no-network** TDD test plan, and the DoD.
3. **Spawn the implementer** (`implementer` subagent, `isolation: worktree`). It
   works **only** in its worktree, follows RED → READ → GREEN → LINT → COMMIT →
   CLOSE, writes `dev/plans/runs/0.5.2-<PACK>-output.json` last, returns the
   REPORT. It does **not** spawn subagents and never touches the main checkout.
   TDD is mandatory — RED tests committed on their own first.
4. **Gate on the witness.** Require `0.5.2-<PACK>-output.json` present and commits
   past baseline. If absent or the REPORT reports a blocker → pack FAILED; triage
   (fixer pack or abandon), never merge.
5. **Codex review** (`agent-harness-reference.md` §3.1):
   `codex exec --model gpt-5.5 -c model_reasoning_effort=high --sandbox read-only
   --cd <worktree>` over the diff/files. Promote to
   `dev/plans/runs/0.5.2-<PACK>-review-<ts>.md` with an `## Orchestrator triage`.
   **PASS → merge; CONCERN → fix or override-with-rationale; never override BLOCK
   (→ fixer pack + re-review).**
6. **Merge → close → clean.** After PASS (or cleared fixes): merge worktree →
   target branch (verify green target tests + no-network suite, run in background
   or `| tail -5`, never the full suite in foreground), write the pack's CLOSED
   block on the STATUS board with the promoted verdict + commit shas, remove the
   worktree.
7. **HITL halt where the ladder says so** (below).

Update `dev/plans/runs/STATUS-0.5.2.md` on **every** transition (one docs commit
per transition). Keep the UN-21/UN-22 acceptance scoreboard current.

- **At kickoff** — confirm the working branch (`feat/0.5.0-resilience-admin` vs a

- **After NAME-aliases merges** — model ids are a client contract. Run a smoke-test
  (per K2; agent unless the operator reserved it — separate dir+port, copied config) of a
  pinned call on a new `aistudio/…`, `vertex/…`, `anthropic/…`, and `openai/…` alias and
  confirm `X-Airlock-Served-By` reports the right provider with no `X-Airlock-Model-Override`.
- **Before removing any legacy alias** — 0.5.2 only **deprecates**; deletion is a
  later release. Do not delete in this release.
- **Release sign-off** — after all packs CLOSED, suite green, and the docs
  smoke-tests pass (`/model/info` + `/v1/models` show capabilities; the
  discover→pin→verify recipe works end-to-end on the test port). Then **finalize per
  runbook §6.1** (smoke per K2 → version + CHANGELOG → local merge + tag per K3; no push).

## Guardrails (do not violate)

- You do **not** edit source/test files in the main checkout — delegate to an
  implementer in a worktree. Docs/STATUS edits on the branch are yours.
- **Production safety ([[airlock-production-safety]]):** never run against the live
  proxy/port. All config/alias smoke-tests use a separate dir+port + copied config.
- Never merge before `output.json` exists and codex has PASSed (or a fix cleared a
  BLOCK and re-review PASSed). Never override a codex **BLOCK**.
- Never `git push`, force-push, or amend without explicit human approval.
- All naming/capability changes are **additive** in 0.5.2 — a pack whose diff
  removes a legacy alias, changes a request-path behavior, or makes `/v1/models`
  drop a standard field is a review **BLOCK**.
- Protect your context: hold conclusions, not raw agent/codex output; delegate
  large reads to `Explore`.
