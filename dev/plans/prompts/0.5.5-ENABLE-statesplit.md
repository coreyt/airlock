## Design Decisions

### A. Module split (by logical cohesion)

| New file | Content from state.py | Lines (approx at b068391) |
|---|---|---|
| `_state_core.py` | `BreakerPolicy`, `_policy_from_mapping`, `configure_breaker`, `policy_for`, `CircuitState`, `ClientState`, `SessionRecord`, `ProviderRateLimitState`, `ClientProviderState`, `ProviderState`, `ModelState`, `StateStore` | 26–38, 41–154, 155–244, 446–856, 856–1512 |
| `_state_spend.py` | `SpendStore`, `ProviderSpend` | 256–444 |
| `_state_mcp.py` | `McpToolState`, `McpServerHealth`, `McpServerState` | 685–784 |
| `_state_persistence.py` | `checkpoint_spend`, `restore_spend`, `checkpoint_state`, `restore_state` | 1513–1644 |

`StateStore` stays in `_state_core.py` (it is the core registry).

### B. state.py becomes a re-export facade

After the split, `state.py` contains:
1. The re-export chain (wildcard re-exports from each sub-module, plus explicit
   names for anything that needs to remain discoverable via `__all__`).
2. `get_store()`, `set_store()`, `_StoreProxy`, and `store = _StoreProxy()` from
   ENABLE-stateprovider (these stay in state.py, not moved to a sub-module).
3. The `tail_jsonl` function (TUI-specific; stays in state.py).
4. The `normalize_client_id` / `NO_CLIENT_ID` re-exports.
5. The tuning constants (`WINDOW_SECONDS`, `MAX_SAMPLES`, etc.).

The facade pattern: each sub-module imports what it needs from the others, and
`state.py` re-exports the full surface with explicit names (not `import *`) so
that IDEs, linters, and `from airlock.fast.state import X` callers all continue
to resolve correctly.

### C. Circular import guard

`_state_core.py` imports only stdlib and `airlock.client_identity`. `_state_spend.py`
imports `litellm` (lazy) and the constants from `_state_core.py`. `_state_mcp.py`
imports from `_state_core.py` only. `_state_persistence.py` imports from
`_state_core.py` and `_state_spend.py`. No circular imports.

`StateStore` (in `_state_core.py`) references `SpendStore` and `McpServerState` —
these must be imported at the top of `_state_core.py` from `_state_spend.py` and
`_state_mcp.py`. Verify no cycles with `python -c "from airlock.fast import state"`.

### D. Underscore prefix = internal

The new files are named with a leading underscore (`_state_*.py`) to signal they
are internal to the `fast/` package. Callers outside `fast/` use `from airlock.fast.state import X` and must not import from `_state_*.py` directly. The underscore does not hide the modules; it is a convention signal only.

### E. No `__all__` expansion required in sub-modules

Sub-modules do not need `__all__`. Only `state.py`'s explicit re-exports matter
for external callers.
