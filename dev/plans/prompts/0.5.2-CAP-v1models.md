## Design Decisions (already resolved — you cannot infer these)

### What to add (design §2.6, §4.4) — PURELY ADDITIVE
LiteLLM serves `GET /v1/models` and `GET /models` (`litellm/proxy/proxy_server.py:8191,8194`)
as the standard OpenAI list: `{"data": [{"id": <model_name>, "object": "model", "created":…,
"owned_by":…}, …], "object": "list"}`. Your seam adds, to **each** `data[i]`, an
`"airlock"` sub-object = `capability_record(entry)` for the config entry whose
`model_name == data[i]["id"]`:
```json
{"id": "aistudio/gemini-3.5-flash", "object": "model", "created": …, "owned_by": …,
 "airlock": {"airlock_provider": "gemini", "endpoints": ["chat","batch"],
             "underlying": "gemini/gemini-3.5-flash", "region": null, "deprecated": false}}
```
- **Additive only.** Never remove/rename a standard field (`id`/`object`/`created`/
  `owned_by`); only ADD `airlock`. A client ignoring `airlock` sees an unchanged response.
- If a model id has no matching config entry (unknown), leave that item unchanged (no
  `airlock` key) — do not crash, do not guess.

### The capability map
Build `{model_name: capability_record(entry)}` from the config `model_list` **once at
install time** and hand it to the middleware instance (pure data; no per-request config
read). Load the config the same way `model_alias.load_from_config` does
(`AIRLOCK_CONFIG` env → default `config.yaml`; tolerate missing/invalid → empty map, log
a warning, seam becomes a transparent pass-through). Reuse `capability.capability_record`.

### The ASGI response-transform middleware
A pure ASGI class (mirror `BatchGatewayMiddleware`'s shape, but transform the RESPONSE):
```python
class ModelsCapabilityMiddleware:
    def __init__(self, app, capability_map): self.app, self.capability_map = app, capability_map
    async def __call__(self, scope, receive, send):
        if scope.get("type") != "http" or scope.get("method") != "GET" \
           or scope.get("path") not in ("/v1/models", "/models"):
            return await self.app(scope, receive, send)        # pass through untouched
        # else: buffer the response, augment JSON, re-send
```
Implementation requirements (this is the load-bearing, fiddly part — get it right):
- Wrap `send`: capture the `http.response.start` (status + headers); buffer all
  `http.response.body` chunks until `more_body` is false.
- **Only transform when** status == 200 AND the `content-type` is JSON AND the parsed
  body is a dict with a list `data`. Otherwise **send the original bytes through
  unchanged** (non-200 error, non-JSON, unexpected shape → never corrupt it).
- Augment: for each item in `data` that is a dict with an `id` in `capability_map`, set
  `item["airlock"] = capability_map[item["id"]]`. Re-serialize with `json.dumps`.
- **Fix the headers:** set `content-length` to the new body length (and drop any
  `transfer-encoding`); preserve all other headers and the status. Header keys are
  lowercase bytes in ASGI — handle case-insensitively.
- Stream-safety: if `more_body` arrives, keep buffering; only emit after the final chunk.
- Never let an exception in augmentation break the response — on any parse/transform
  error, fall back to sending the original bytes (log at debug). Robustness over cleverness.
