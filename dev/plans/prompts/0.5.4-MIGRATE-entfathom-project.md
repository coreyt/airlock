## Design Decisions (already resolved)

### `project_enterprise(event: RequestEvent) -> dict`
Reproduces `_build_record`'s output dict **field-for-field** from the event. Mirror
`_build_record` (:483-539) exactly, sourcing each field from the event instead of
re-deriving:
- `timestamp` → `event.timestamp`; `record_type` → `"request"`; `success`, `model`,
  `user`, `team`, `request_id`(=`event.request_id`), `error`/`error_type`/
  `failure_category`, `airlock_provider`, `start_time`/`end_time`, `duration_ms`.
- `messages` → `event.messages`; `response` → `_serialize(event.response_obj) if
  event.response_obj else None` (reuse enterprise `_serialize`).
- spread `**event.usage`, then `**event.mcp_meta`, then `**event.guardrail_meta`
  (same key order as the builder, :508-510).
- `mutations` → `event.mutations`; `served` → `event.served`; `attribution` →
  `event.attribution`; `airlock_client` → `event.airlock_client`.
The ONLY accepted difference vs the live builder is `timestamp` (the registered
convergence, §3.4) — the golden test compares **ignoring `timestamp`**.

### `project_fathom(event: RequestEvent) -> dict`
Reproduces `_fathom_properties` output (:121-172) from the event — the env-flag
gating stays here. Use fathom's helpers (`_json_text`, `_response_text`, `_env_flag`,
imported from `airlock.callbacks.fathom_logger`). Base properties from the event:
`timestamp`, `success`, `error_flag`(=`not event.success`), `model`,
`airlock_provider`, `request_id`, `call_id`(=`event.request_id`),
`prompt_tokens`/`completion_tokens`/`total_tokens` (from `event.usage`, default 0),
`cost` (= `event.response_cost if event.response_cost is not None else 0` — matches
`kwargs.get("response_cost", 0)`), `duration_ms`, `failure_category`,
`call_type`/`mcp_tool_name`/`mcp_server_name` (from `event.mcp_meta`, `.get` → None
if absent). Then the env-gated additions, sourced from the event:
- `AIRLOCK_FATHOM_STORE_CLIENT` → `airlock_client` = `event.airlock_client`
- `AIRLOCK_FATHOM_STORE_USER_TEAM` → `user`/`team`
- `AIRLOCK_FATHOM_STORE_ERROR_DETAILS` → `error_type`/`error`
- `AIRLOCK_FATHOM_STORE_MESSAGES` → `messages_json` = `_json_text(event.messages)`
- `AIRLOCK_FATHOM_STORE_RESPONSE_TEXT` → `response_text` = `_response_text(event.response_obj)`
- `AIRLOCK_FATHOM_STORE_HEADERS` → `headers_json` = `_json_text(event.request_headers)`
- `AIRLOCK_FATHOM_STORE_MCP_PAYLOADS` → `mcp_arguments_json` = `_json_text(event.mcp_arguments)`
Finally apply the **same None-filter**: `return {k:v for k,v in properties.items() if
v is not None}`.

> Note: `_fathom_properties` reads `record.get("call_type")` from the base record;
> `call_type` only exists in the base when `mcp_meta` populated it, so it comes from
> `event.mcp_meta.get("call_type")`. Mirror that exactly (absent → None → filtered).

## Golden equivalence tests (`tests/test_projections_equiv.py`) — define success (NO network)

start_time, end_time, success)` fixtures covering: a plain success; a provider
failure (`success=False`, `kwargs["exception"]` set); a pre-call-shaped failure; an
MCP call (`call_type=="call_mcp_tool"` + `mcp_arguments`); a Gemini success; a request

each fixture:

   `timestamp` from both** (the one accepted convergence). All other keys/values
   identical.

   (at least: all-off, all-on, and a couple of mixed) via `monkeypatch.setenv`, call
   the live `_fathom_properties(*inputs, success=...)` → `expected` and
   `project_fathom(build_request_event(*inputs, success=...))` → `got`; assert `got ==
   expected` after deleting `timestamp` from both.

> and idempotent, so order shouldn't matter, but assert it).
