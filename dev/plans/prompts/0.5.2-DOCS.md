### The naming convention (routing.md)
- Rule: **prefix = provider.** `anthropic/`, `openai/`, `aistudio/` (Google AI Studio,
  served as `gemini`), `vertex/` (Google Vertex, served as `vertex_ai`), `mistral/`,
  `perplexity/`, `tavily/`, `vllm/` (local, openai-compatible). The bare name (e.g.
  `gemini-3.5-flash`) is a documented, ops-repointable **default** (AI Studio for Gemini);
  the **prefixed name is the stable client contract**.
- `aistudio/` and `vertex/` are deliberately distinct from LiteLLM's native `gemini/` /
  `vertex_ai/` tokens so the alias is exact-matched, never re-parsed as provider routing.
- `smart` is a routing directive, not a provider model тАФ never pinned, no prefix.

### Discover тЖТ pin тЖТ verify recipe (routing.md тАФ the centerpiece)
1. **Discover:** `GET /v1/models` (or `/model/info`) тЖТ read each model's `airlock` object:
   `airlock_provider`, `endpoints`, `region`, `deprecated`.
2. **Pin:** send `model: "aistudio/gemini-3.5-flash"` (or `vertex/тАж`) тАФ a concrete
   prefixed name is auto-pinned (fallbacks/retries off тЖТ 429 on overload, never a silent
   model swap).
3. **Verify:** read `X-Airlock-Served-By` on the response (`gemini` for AI Studio,
   `vertex_ai` for Vertex, `anthropic`/`openai`/тАж); it equals the `airlock_provider` you
   discovered. (`X-Airlock-Served-Region` appears for gateway/region backends.)
