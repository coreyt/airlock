- **`is_batch_call(data, call_type="") -> bool`** mirrors `is_mcp_call`. It is

  (e.g. `acreate_batch`, `aretrieve_batch`, `acreate_file`/`create_file`,
  `afile_content`) **or** the batch markers are present in `data`

  litellm call_type strings to fix the exact set; if a string is uncertain,
  match on the data markers too (defense in depth) and note it.
- **Guardian gating**: in `async_pre_call_hook`, wrap the model-specific block
  (alias resolution, provider protection, `infer_provider`, circuit breaker,
  failover) behind `if not is_batch_call(data, call_type):` ÔÇö exactly mirroring
  the existing `if not mcp:` skip. **Keep** client-level rate-limit / threat
  checks running on batch calls. **Keep** the existing reactive null-guards as
  defense-in-depth (do not remove them).
- **Trust boundary**: ignore caller-supplied guardrail/metadata disables on the
  batch path.
