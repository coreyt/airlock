## Design Decisions (already resolved — do not redesign)

1. **Where.** `AirlockModelOverrideHeaders.async_post_call_response_headers_hook(self,
   data, user_api_key_dict, response, request_headers=None) -> dict[str,str] | None`
   fires for **both** streaming and non-streaming success (litellm
   `common_request_processing.py:1276`, headers flushed before the SSE body). It
   already (a) reads `metadata = data.get("metadata", {})`, (b) builds a
   `response_headers` dict (currently Gemini headers + any `airlock_response_headers`),
   (c) updates `response._hidden_params["additional_headers"]`, (d) returns the dict (or
   `None`). You ADD served-by emission into this same flow — keep the existing behavior
   intact and additive.
2. **Attribute.** Compute `served = attribute_served_backend(response)` (NO
   `cost_fallback` here — cost is finalized later in OBS-log; `response_cost` is `None`
   at header-flush time on streams, that's expected). The helper already resolves the
   provider as `_hidden_params["custom_llm_provider"]` **or** the wrapper attribute
   `getattr(response, "custom_llm_provider", None)` (CC-T3, streaming-correct) — you do
   NOT re-implement that; just call it.
3. **Emit (config-gated, default-on).** If `get_transparency_config().served_headers`
   is truthy, merge `served_headers(served)` into the returned header dict (and into
   `response._hidden_params["additional_headers"]`, mirroring how the existing headers
   are surfaced so they actually reach the client on both paths). `served_headers`
   returns `{}` when the provider is unknown → **header simply omitted, never guessed**
   (CC-T3). If `served_headers` config is false → emit nothing (opt-out, CC-T7).
4. **Stash identity into metadata (best-effort).** Store the identity part of `served`
   into `metadata` under `metadata["airlock_served"]` (e.g. a dict of provider/
   api_base_host/region/model_id/backend_kind — NOT response_cost). This is a
   best-effort optimization for downstream; **OBS-log does NOT depend on it** (it calls
   `attribute_served_backend` itself per Decision 5). Use the same `metadata` object the
   hook already reads (`data.get("metadata")`); if it's missing, `data.setdefault("metadata", {})`.
5. **No request-path change (CC-T4).** This is post-call, observe-only header emission +
   a startup config call. It must not alter whether/where any request goes, nor change
   the response body.
6. **Startup wiring.** In `proxy.py`, import `configure_transparency` from
   `airlock.transparency` alongside the other `configure_*` imports (~`:379-383`) and
   call `configure_transparency(config)` in the startup sequence after
   `configure_guardrail_overrides(config)` (~after `:394`), using the same `config`
   dict variable. Without this, `get_transparency_config()` returns defaults — which is
   safe, but wiring makes operator opt-outs effective.
