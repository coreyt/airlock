## Design Decisions (already resolved — you cannot infer these)

1. **Distinguish "refused" from "simply unknown".** `resolve()` returns `None` both for
   a dropped-qualifier refusal (`gpt-5.6-mini`) AND for a name with no near match at
   all. The 404-with-suggestions is for the refusal case. Detect it by calling
   `suggest(model_name)` (model_alias.py:372) — a **non-empty** suggestion list means
   there was a near match worth proposing. An empty list → let it fall through to
   litellm's own error (do not fabricate an empty suggestions block).

2. **Short-circuit at the resolve site.** In guardian.py, at the `resolve()` call
   (~line 316), when `resolved is None` and `suggest()` is non-empty, raise a typed
   exception rendered as **404 + `code: "model_not_found"`**, OpenAI-shaped. Do NOT
   leave the unknown name in `data["model"]` to reach litellm.

3. **The error body must be self-sufficient in `error.message`.** Most SDKs surface
   only that string. It must name the rejected model, the top suggestion, and the
   alternatives — readable without parsing the structured block. Add the machine form
   under `error.airlock.suggestions` (the list from `suggest()`: `{model, score, tier}`).
   Follow the body shape of `block_response_payload` (proxy_errors.py:72-92) but at
   status 404, not 429. Add a new typed exception + FastAPI handler paralleling
   `AirlockProviderBlocked` / its handler, and register it in
   `install_airlock_error_handlers_on_proxy_app()` (proxy_errors.py:110).

4. **`X-Airlock-Model-Suggestion` header.** Use the established `;`-joined `key=value`
   grammar. **`_mutation_token` (transparency.py:261) is Mutation-specific — do NOT call
   it.** Instead add a small sibling serializer in transparency.py that reuses
   `_header_safe` (transparency.py:256) and the same `";".join(...)` + byte-budget
   discipline as `mutations_header` (transparency.py:274-293). Emit e.g.
   `requested=gpt-5.6-mini;suggested=gpt-5.6-luna;reason=dropped_qualifier`.
   If F-1 merged first, place/emit it consistently with its admission header.

5. **Never leak a model outside the caller's catalog.** Filter `suggest()` output
   against what the caller is entitled to before putting it in the body or header. Use
   the client identity from `client_identity` and whatever per-client catalog filtering
   the codebase already applies (search for it; if none exists, the full configured
   catalog is the entitlement, and record that finding). A helpful error must never
   become a capability-disclosure channel.

6. **No routing regression.** Names that resolve today MUST still resolve. The
   short-circuit fires only on the `None` + non-empty-suggestions path.
