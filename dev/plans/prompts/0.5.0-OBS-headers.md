## Design Decisions (resolved — do not redesign)

### Part A — `X-Airlock-Mutations` header (must-have, default-on)
- In `async_post_call_response_headers_hook`, after the existing served-headers block and
  before the final `return dict(response_headers)`, add:
  ```python
  cfg = get_transparency_config()
  ledger = metadata.get("airlock_mutations") or []
  if ledger and cfg.mutation_headers != "off":
      header_val = mutations_header(ledger, cfg.mutation_header_budget_bytes)
      if header_val:
          response_headers["X-Airlock-Mutations"] = header_val
  ```
  (Reuse the single `get_transparency_config()` call the served block already makes, if
  present — don't call it twice needlessly.) `mutations_header` already enforces CC-T2
  (no content for non-allowlisted fields) and the byte bound — you just call it.
- CC-T6 streaming: the ledger at header-flush time naturally contains only the
  pre/during-call mutations (post-call reasoning-strip hasn't run yet on streams); the full
  ledger is in the JSONL log (OBS-log). Do NOT special-case — serialize the ledger as-is.
- This fires for streaming + non-streaming alike; the header is additive (CC-T7).

### Part B — `X-Airlock-Explain` body envelope (opt-in, NON-STREAMING ONLY)
- Add `async_post_call_success_hook(self, data, user_api_key_dict, response)` to the same
  class. This hook fires for **non-streaming** success (streaming uses the separate
  `async_post_call_streaming_iterator_hook`, which you do NOT implement — so the envelope is
  automatically non-streaming-only, satisfying Decision 7/CC-T5).
- Read the opt-in from the **request headers**, which litellm stashes in
  `data["proxy_server_request"]["headers"]` (confirmed). Compare case-insensitively against
  `cfg.explain_body_optin_header` (default `"X-Airlock-Explain"`); treat a truthy value
  (e.g. `"1"`, `"true"`) as opt-in. If absent/falsy → **return `response` unchanged** (body
  byte-identical, CC-T5/CC-T7 — this is the default path and MUST be a no-op).
- When opted in: attach `airlock.mutations` = the serialized ledger
  (`metadata.get("airlock_mutations") or []`, each `Mutation` via `dataclasses.asdict`) to the
  response body as an additive field, then return `response`. **Investigate empirically how the
  body actually serializes** (reference: resolve behavior by observation, not assumption):
  litellm responses are usually pydantic `ModelResponse` — confirm whether an attached field
  (e.g. setting `response.airlock = {...}`, or via the model's extra/`__dict__`) appears in the
  serialized JSON the client receives. If a plain attribute does not serialize, use the
  mechanism that does (e.g. the model's extra-fields path). Write the test against the SAME
  serialization the proxy uses (e.g. `response.model_dump()` / `json`), so a green test proves
  the client would actually see it. If after a focused investigation no clean, serialization-safe
  attach exists without touching a DO NOT TOUCH file, implement Part A solidly, `[DETECT]` the
  blocker, leave Part B out, and CLOSE with Part A done + the envelope flagged for the
  orchestrator — do NOT force a fragile body mutation.
- The envelope is metadata-only (mutation records — already value-safe per CC-T2); it never
  carries message/prompt content.
