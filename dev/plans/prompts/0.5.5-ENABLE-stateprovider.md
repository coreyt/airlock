## Design Decisions (already resolved — do not re-derive)

### A. Mechanism: transparent proxy + lazy-init default

Replace the bare singleton at `state.py:1645` with:

```python
_default_store: StateStore | None = None

def get_store() -> StateStore:
    """Return the active StateStore (injected or default singleton)."""
    global _default_store
    if _default_store is None:
        _default_store = StateStore()
    return _default_store

def set_store(s: StateStore | None) -> None:
    """Inject (or clear) the active StateStore. Pass None to reset to default."""
    global _default_store
    _default_store = s

class _StoreProxy:
    """Transparent proxy so `from .state import store` callers need no changes.

    All attribute access is forwarded to the active store returned by get_store().
    `set_store(fresh)` re-routes all existing `store.X` calls immediately.
    """
    def __getattr__(self, name: str):
        return getattr(get_store(), name)

    def __repr__(self) -> str:
        return f"<_StoreProxy wrapping {get_store()!r}>"

store = _StoreProxy()
```

### B. No callers need to change

All 56 existing importers do `from .state import store` and then call `store.X(...)`.
Because `store` is now the proxy object (same Python object, always), attribute
lookups go through `__getattr__` → `get_store()` → the active `StateStore`. The
proxy itself is never replaced — only `_default_store` changes.

### C. Test isolation via `set_store` + autouse fixture

Add a `pytest` autouse fixture to `tests/test_fast_state.py` (scope=`function`)
that calls `set_store(StateStore())` before each test and `set_store(None)` after,
so concurrent tests don't share state through the global. **Do not add this fixture
to `conftest.py` yet** — that is scope-creep; ENABLE-statesplit may generalize it.

### D. `tail_jsonl` references `store` as a free variable

`state.py:1658` has `store` referenced inside `tail_jsonl` in a call like
`store.something(...)`. After the change, `store` is still the proxy — tail_jsonl
works unchanged.

### E. `normalize_client_id` and `NO_CLIENT_ID` are re-exports, not attributes of `store`

Lines 23–24 re-export these from `airlock.client_identity`. They are **not** on
`StateStore`; they are module-level names. Callers that do

needed.
