## Design Decisions (resolved)

### 1. Extend `RequestRecorder` (request_event.py) — backward-compatible
- `register(self, sink, *, name, async_only=False)` — store `async_only` on the
  `_Registration` (add the field, default False).
- `dispatch(self, event, *, is_async=True)` — keep the lock+snapshot; for each reg,
  **skip** when `reg.async_only and not is_async`; otherwise call with the SAME
  per-sink try/except isolation. Default `is_async=True` so existing EVENT tests
  (`dispatch(event)`) and existing `register(...)` calls are unaffected.

### 2. `RequestRecorderCallback(CustomLogger)` (request_event.py) — the single seam callback
Holds a `RequestRecorder`. A private helper builds once + dispatches:
```python
def _record(self, kwargs, response_obj, start_time, end_time, *, success, is_async):
    event = build_request_event(kwargs, response_obj, start_time, end_time, success=success)
    self._recorder.dispatch(event, is_async=is_async)
```
- `log_success_event(...)`  → `self._record(..., success=True,  is_async=False)`
- `log_failure_event(...)`  → `self._record(..., success=False, is_async=False)`
- `async_log_success_event(...)` → `await asyncio.to_thread(self._record, kwargs, response_obj, start_time, end_time, success=True, is_async=True)` (mirror the loggers' to_thread offload; pass success/is_async via functools.partial or a small wrapper since to_thread takes positional args)
- `async_log_failure_event(...)` → same with `success=False, is_async=True`
The sync paths pass `is_async=False` (fathom, an async_only sink, is skipped there —
preserving fathom's async-only firing surface, §5a).

### 3. `AirlockLogger.record_event(self, event: RequestEvent) -> None` (enterprise_logger.py)
The enterprise SINK. Reproduces today's `log_success_event`/`log_failure_event` output
using the projection:
```python
def record_event(self, event):
    from airlock.callbacks.projections import project_enterprise
    record = project_enterprise(event)
    _write_log(record)
    if event.success:
        logger.info("request_logged model=%s user=%s tokens=%s",
                    record["model"], record.get("user"), record.get("total_tokens"))
    else:
        logger.warning("request_failed model=%s user=%s client=%s category=%s error_type=%s error=%s",
                       record["model"], record.get("user"), record.get("airlock_client"),
                       record.get("failure_category"), record.get("error_type"), record.get("error"))
```
(Match the existing log-line formats at enterprise_logger.py:358-363 and :384-392 exactly.)

### 4. `AirlockFathomLogger.record_event(self, event: RequestEvent) -> None` (fathom_logger.py)
The fathom SINK. Reproduces `_log_event`'s functional path (skip / engine / call_id /
dedup / write) using `project_fathom`:
```python
def record_event(self, event):
    from airlock.callbacks.projections import project_fathom
    if event.guardrail_meta.get("airlock_skip_fathom_logger"):   # airlock_-prefixed -> rides on guardrail_meta
        return
    db_engine = self._get_engine()
    if not db_engine or WriteRequestBuilder is None:
        return
    call_id = event.request_id or uuid.uuid4().hex
    if self._should_skip_call_id(call_id):
        return
    builder = WriteRequestBuilder("airlock_log")
    builder.add_node(row_id=uuid.uuid4().hex, logical_id=call_id, kind="RequestLog",
                     properties=project_fathom(event), source_ref="airlock:fathom_logger", upsert=True)
    try:
        db_engine.write(builder.build())
    except Exception as e:
        logger.error(f"FathomDB write failed: {e}")
```
(Pure `_debug_enabled()` logging branches may be omitted — they are debug-only, no
functional effect. Keep the skip/engine/call_id/dedup/write logic identical.)

## Target tests (`tests/test_recorder_wiring.py`) — NO network
1. **Recorder async_only**: register an always sink + an `async_only=True` sink;
   `dispatch(event, is_async=False)` runs only the always sink; `dispatch(event,
   is_async=True)` runs both. Registration order preserved.
2. **Backward compat**: `dispatch(event)` (no is_async) behaves as is_async=True; an
   existing `register(sink, name=...)` (no async_only) still always runs.
3. **RequestRecorderCallback**: with a fake recorder capturing `(event, is_async)`:
   `log_success_event` → one dispatch `is_async=False, event.success True`;
   `log_failure_event` → `is_async=False, success False`; the async variants →
   `is_async=True` (await them). Build inputs in-process.
4. **enterprise.record_event**: monkeypatch `_write_log`; assert it's called with
   exactly `project_enterprise(event)`, for both success and failure events.
5. **fathom.record_event**: (a) `airlock_skip_fathom_logger` in metadata → no write;
   (b) no engine → no write; (c) dedup: same `call_id` twice → second is skipped;
   (d) normal → writes `project_fathom(event)` (stub the engine + WriteRequestBuilder
   or assert the properties passed). 
6. **Dormancy**: importing `airlock.callbacks.recorder` creates `recorder_callback`
   but does NOT add it to `litellm.logging_callback_manager`'s lists (assert the
   recorder_callback instance is absent from the success/failure callback lists).
