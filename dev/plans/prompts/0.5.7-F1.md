## Design Decisions (already resolved — you cannot infer these)

1. **The admitted-path header** is additive and simple. In
   `async_post_call_response_headers_hook` (model_override_headers.py:28), read
   `metadata.get("airlock_admission")` and, when present with `action == "admitted"`,
   set `response_headers["X-Airlock-Admission"] = "admitted"`. This metadata is NOT
   under `airlock_response_headers`, so it will NOT surface automatically — the
   explicit read is required.

2. **The shed path is the real work.** Today guardian.py:305-307 raises a plain
   `ValueError`, which does NOT go through `proxy_errors.py`, so it gets no 429 and no
   `Retry-After` header — only text in the message. AC-2 requires a real 429 with both
   headers. **Resolution:** introduce a typed exception paralleling
   `AirlockProviderBlocked` (proxy_errors.py:42) — e.g. `AirlockAdmissionShed` — and a
   matching FastAPI handler that renders `JSONResponse(status_code=429, ...)` with
   `X-Airlock-Admission: shed; retry_after=N` **and** `Retry-After: N`. Register it in
   `install_airlock_error_handlers_on_proxy_app()` (proxy_errors.py:110) alongside the
   existing one. Replace the `raise ValueError(...)` at guardian.py:305 with a raise of
   the new exception, carrying `retry_after`.

3. **Retry-After is a whole number ≥ 1.** Reuse `retry_after_seconds()`
   (proxy_errors.py:67). The body value may keep the float `retry_after`; the header
   must be the whole-second form. `X-Airlock-Admission: shed; retry_after=N` uses the
   same integer N.

4. **Off-by-default invariant.** When `admission.enabled: false` the gate is None
   (guardian.py:286 guard), so neither header is ever set. Do not add a code path that
   emits them when the gate is absent.

5. **OpenAI-compatible body.** The shed 429 body follows the existing
   `block_response_payload` shape (proxy_errors.py:72-92): `{"error": {...}}` with an
   `airlock` sub-object. Reuse `sanitize_reason()` for any echoed text.
