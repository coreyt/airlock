### C. `RequestRecorder` (the seam — design §5/§5a):
- A registry of **sinks**. A sink is a callable `(RequestEvent) -> None` registered
  with a stable `name`. `register(sink, *, name)` appends to an **ordered** list;
  dispatch order == registration order (deterministic).
- `dispatch(event: RequestEvent) -> None`: iterate sinks in order; wrap **each** sink
  call in `try/except Exception` that logs (`logger.warning(..., exc_info=True)`) and
  **continues** — a raising sink must **never** propagate to the caller or stop other
  sinks (AC-SEAM, §5 "never raise from telemetry"). An **empty** recorder dispatch is
  a no-op. `dispatch` itself never raises.
- Provide a way to introspect registered sink names/order for tests (e.g. a
  `sinks`/`sink_names` property). Do NOT install into the live `litellm`
  callback manager in this pack.

### D. `build_request_event(kwargs, response_obj, start_time, end_time, *, success) ->
RequestEvent` — the single producer. Import and reuse the enterprise helpers
(`from airlock.callbacks.enterprise_logger import _normalize_failure,
_get_airlock_client, _serialize`) and the same upstream helpers it imports
(`infer_provider`, `attribute_served_backend`, `is_gemini_provider`,
`classify_gemini_response`, `build_gemini_response_headers`). Guard `litellm_params`/
`metadata` with `or {}` exactly as `_build_record` does.
